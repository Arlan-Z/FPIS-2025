# "Lesson 1: Field projects for Information Systems"

1.  **Зачем нам вообще изучать Информационные Системы (ИС)? (Слайд 3)**
    *   **Кто использует ИС?** Многие специалисты: менеджеры по продажам, руководители, финансовые консультанты и т.д.
    *   **Польза ИС:**
        *   Это незаменимые инструменты для достижения карьерных целей.
        *   Бизнесу ИС помогают **увеличивать доходы и сокращать расходы**.

2.  **Ключевые понятия: Данные vs. Информация (Слайды 4, 6)**
    *   **Данные (Data):** Это сырые, необработанные факты, цифры, символы. Сами по себе они могут ничего не значить.
        *   *Примеры:* "100", "Яблоко", "01.01.2023", фотография товара, аудиозапись разговора.
        *   *Типы данных (Слайд 6):*
            *   **Буквенно-цифровые:** Текст, числа.
            *   **Графические:** Картинки, схемы.
            *   **Аудио:** Звуки, речь.
            *   **Видео:** Движущиеся картинки.
    *   **Информация (Information):** Это обработанные, осмысленные данные, которые полезны для принятия решений. Информация – это один из **самых ценных ресурсов** организации.
        *   *Пример:* Если данные – "100 яблок продано", "цена 1 яблока – 50 руб.", то информация – "Выручка от продажи яблок составила 5000 руб.".

3.  **Что такое Информационная Система (ИС)? (Слайды 4, 10)**
    *   **Определение (Слайд 4):** Это набор связанных между собой компонентов, которые:
        1.  **Собирают** данные и информацию.
        2.  **Обрабатывают** (манипулируют) их.
        3.  **Распространяют** (выдают) результат.
        4.  Обеспечивают **обратную связь** (позволяют корректировать работу системы).
        ...и все это для достижения какой-то **цели**.
    *   **Процесс работы ИС (Слайд 10):**
        1.  **Ввод (Input):** Сбор сырых данных (например, ввод заказа в систему).
        2.  **Обработка (Processing):** Превращение данных в полезную информацию (например, расчет суммы заказа, проверка наличия товара).
        3.  **Вывод (Output):** Выдача полезной информации (например, счет на оплату, отчет о продажах).
        4.  **Обратная связь (Feedback):** Информация из системы, которая используется для изменения ввода или обработки (например, если товара нет на складе, система предлагает заказать его или сообщает об ошибке).

4.  **Роль ИС в современном бизнесе (Слайд 5):**
    *   Создание **новых продуктов, услуг и бизнес-моделей**.
    *   Улучшение **взаимодействия с клиентами и поставщиками**.
    *   **Улучшение принятия решений** (за счет качественной информации).
    *   Достижение **операционного совершенства** (работать эффективнее, быстрее, дешевле).
    *   Получение **конкурентного преимущества** (быть лучше других на рынке).

5.  **Характеристики ценной информации (Слайды 7, 8):**
    Чтобы информация была действительно полезной, она должна быть:
    *   **Доступной (Accessible):** Легко получить, когда нужно.
    *   **Точной (Accurate):** Без ошибок. Важно: "мусор на входе – мусор на выходе" (GIGO – Garbage In, Garbage Out) – если в систему ввести неверные данные, то и информация будет неверной.
    *   **Полной (Complete):** Содержать все важные факты.
    *   **Экономичной (Economical):** Стоимость получения информации не должна превышать ее ценность.
    *   **Гибкой (Flexible):** Можно использовать для разных целей.
    *   **Релевантной (Relevant):** Важной именно для текущей задачи или решения.
    *   **Надежной (Reliable):** Можно доверять (источник, метод сбора).
    *   **Защищенной (Secure):** Недоступной для тех, кому она не предназначена.
    *   **Простой (Simple):** Понятной, не слишком сложной.
    *   **Своевременной (Timely):** Предоставленной вовремя (вчерашняя погода не поможет выбрать одежду сегодня).
    *   **Проверяемой (Verifiable):** Можно подтвердить ее правильность из других источников.

6.  **Ценность информации (Слайд 9):**
    *   Главная ценность информации в том, что она **помогает принимать правильные решения**.

7.  **Компоненты (элементы) компьютерной ИС (CBIS - Computer-based Information System) (Слайды 11, 13, 14):**
    Любая компьютерная информационная система состоит из:
    *   **Аппаратное обеспечение (Hardware):** Физические устройства – компьютеры, серверы, принтеры, клавиатуры и т.д.
    *   **Программное обеспечение (Software):** Программы, приложения.
    *   **Базы данных (Database):** Организованное хранилище данных.
    *   **Телекоммуникации, сети, Интернет (Telecommunication, Networks, and the Internet):** Средства связи и передачи данных.
    *   **Люди (People):** Пользователи, разработчики, администраторы системы.
    *   **Процедуры (Procedures):** Правила, инструкции по работе с системой.
    *   Эти компоненты можно сгруппировать (Слайд 11) на:
        *   **Прикладные (Application):** Что система делает (ПО, БД).
        *   **Технические (Technical):** На чем работает (железо, сети).
        *   **Организационные (Organizational):** Кто и как использует (люди, процедуры).

8.  **Типы ИС (Слайд 12, 15):**
    *   **По способу реализации (Слайд 12):**
        *   **Ручные (Manual):** Без компьютеров (например, картотека в библиотеке).
        *   **Компьютерные (Computer-based IS - CBIS) или Управленческие ИС (Management IS - MIS):** С использованием компьютеров. Именно о них в основном и идет речь.
    *   **По масштабу использования (Слайд 15):**
        *   **Персональные ИС (Personal IS):** Для одного пользователя (например, ваш список дел в телефоне).
        *   **Групповые ИС (Group IS):** Для небольшой группы людей, отдела (например, общий календарь команды).
        *   **Корпоративные ИС (Enterprise IS):** Для всей организации (например, система управления ресурсами предприятия - ERP).

# Lesson 2 part 1: Data Management

1.  **Что такое База Данных (БД) и зачем она нужна? (Слайды 3, 4, 6)**
    *   **База Данных (Database):** Это как большая, хорошо организованная электронная картотека (или шкаф с папками), где хранятся связанные между собой данные. Эти данные упорядочены так, чтобы их было легко находить и использовать.
        *   *Проще говоря:* Упорядоченный набор данных, обычно в электронном виде, описывающий что-то реальное (студентов, товары, заказы).
    *   **Система Управления Базами Данных (СУБД / DBMS - Database Management System):** Это специальная программа (или комплекс программ), которая помогает создавать, управлять и работать с базами данных. Она как библиотекарь, который знает, где какая "книга" (данные) лежит, и помогает ее найти или положить на место.
        *   *Проще говоря:* Софт, который "заведует" базой данных и позволяет пользователям и другим программам с ней общаться.
    *   **Администратор Базы Данных (АБД / DBA - Database Administrator):** Это специалист, который отвечает за все, что связано с базой данных в организации – от ее создания и настройки до обеспечения безопасности и производительности.
        *   *Проще говоря:* "Главный по тарелочкам" в мире баз данных.
    *   **Зачем управлять данными?** Без данных и возможности их обрабатывать организация не сможет нормально работать. Чтобы превратить "сырые факты" (данные) в полезную информацию, их нужно сначала организовать – то есть, создать базу данных и управлять ею с помощью СУБД.

2.  **Иерархия данных: из чего строятся данные (Слайды 7-11)**
    Представь, что данные – это конструктор LEGO.
    *   **Бит (Bit):** Самая маленькая деталька – 0 или 1 (включено/выключено).
    *   **Байт (Byte):** Группа из 8 битов. Обычно 1 байт = 1 символ (буква, цифра).
    *   **Символ (Character):** Буква, цифра, знак препинания (например, 'А', '5', '?').
    *   **Поле (Field) / Атрибут (Attribute) / Столбец (Column):** Это характеристика чего-либо. Например, "Имя", "Возраст", "Цена". Состоит из символов.
        *   *Пример:* В таблице студентов поле "Имя" может содержать "Иван".
    *   **Запись (Record) / Экземпляр (Instance) / Строка (Row):** Это набор полей, описывающий один объект. Например, вся информация об одном студенте (его Имя, Фамилия, Возраст, Группа).
        *   *Пример:* Строка в таблице: "Иван", "Петров", 20, "Группа 101".
    *   **Файл (File) / Таблица (Table):** Набор однотипных записей. Например, список всех студентов.
        *   *Пример:* Таблица "Студенты", где каждая строка – это один студент, а каждый столбец – его характеристика.
    *   **База Данных (Database):** Набор связанных между собой файлов (таблиц). Например, база данных университета может содержать таблицу "Студенты", таблицу "Преподаватели", таблицу "Курсы".

3.  **Сущности, Атрибуты и Элементы данных (Слайды 13, 14)**
    *   **Сущность (Entity):** То, о чем мы храним информацию. Это могут быть люди, места, объекты, события.
        *   *Примеры:* Студент, Преподаватель, Книга, Заказ.
        *   Сущности могут быть связаны друг с другом (например, Студент изучает Курс).
    *   **Атрибут (Attribute):** Характеристика (свойство) сущности. То же, что и **Поле**.
        *   *Примеры:* Для сущности "Студент" атрибутами могут быть: Номер студенческого, Имя, Фамилия.
    *   **Элемент данных (Data item):** Конкретное значение атрибута для конкретного экземпляра сущности.
        *   *Пример:* Для атрибута "Имя" сущности "Студент" элементом данных может быть "Мария".

4.  **Ключи в Базах Данных (Слайды 15, 16): Зачем они нужны?**
    Ключи – это специальные поля (или наборы полей), которые помогают уникально идентифицировать записи и связывать таблицы между собой.
    *   **Первичный ключ (Primary Key):** Поле (или комбинация полей), которое **уникально** определяет каждую запись в таблице. Не может быть пустым и не может повторяться.
        *   *Пример:* "Номер студенческого билета" в таблице "Студенты". У каждого студента он свой.
    *   **Составной первичный ключ (Compound Primary Key):** Если одного поля недостаточно для уникальной идентификации, используется комбинация из нескольких полей.
        *   *Пример:* В таблице "Оценки" первичным ключом может быть комбинация "Номер студента" + "Код курса" (потому что один студент может иметь оценки по разным курсам, и один курс могут изучать разные студенты).
    *   **Вторичный ключ (Secondary Key):** Поле, по которому часто ищут данные, но оно не обязательно уникальное. Используется для ускорения поиска.
        *   *Пример:* "Фамилия" в таблице "Студенты". Может быть несколько студентов с одинаковой фамилией.
    *   **Внешний ключ (Foreign Key):** Поле в одной таблице, которое ссылается на первичный ключ в другой таблице. Используется для связи таблиц.
        *   *Пример:* В таблице "Оценки" поле "Номер студента" будет внешним ключом, ссылающимся на первичный ключ "Номер студенческого билета" в таблице "Студенты". Это показывает, какому студенту принадлежит оценка.

5.  **Система баз данных (приложение) и Управленческая Информационная Система (MIS) (Слайд 17)**
    *   **Система баз данных (Database Application System):** Это комбинация самой БД, СУБД и прикладных программ, которые обращаются к этой БД.
    *   **Управленческая Информационная Система (MIS):** Это система баз данных плюс все остальные компоненты ИС (железо, софт, сети, люди, процедуры), которые вместе превращают сырые данные в полезную информацию для управления.

6.  **Два подхода к хранению данных: Файловый vs. Базы данных (Слайды 18-27)**
    Это очень важное сравнение "старого" и "нового" подходов.
    *   **Файловый подход (старая школа):** (Слайды 18, 19, 21, 22)
        *   **Как это работает:** Каждое приложение (программа) имеет свой собственный набор файлов с данными. Данные часто дублируются в разных файлах.
        *   **Проблемы:**
            *   **Неконтролируемое дублирование данных (Uncontrolled Redundancy):** Одна и та же информация хранится в нескольких местах. Это занимает лишнее место и усложняет обновление.
            *   **Несогласованность данных (Data Inconsistency):** Если изменить данные в одном файле, они могут остаться старыми в другом. Получается путаница.
            *   **Негибкость (Inflexibility):** Системы заточены под конкретные запросы, трудно получить новую, нестандартную информацию.
            *   **Плохое совместное использование данных (Limited Data Sharing):** Каждое приложение "держит" свои данные при себе.
            *   **Зависимость программ от данных (Program-Data Dependency):** Если меняется структура файла с данными, нужно переписывать программу, которая с ним работает. Это дорого и долго.
            *   **Слабое обеспечение стандартов:** Имена данных, форматы могут отличаться в разных файлах.
    *   **Подход с использованием Баз Данных (современный):** (Слайды 20, 23, 24)
        *   **Как это работает:** Есть **централизованная СУБД**, которая управляет всеми данными. Данные хранятся в одном большом "хранилище" (базе данных) и не раздроблены по отдельным файлам для каждого приложения. СУБД обеспечивает добавление, хранение, обновление и извлечение данных.
        *   **Преимущества (Слайд 25):**
            *   **Уменьшение дублирования данных.**
            *   **Улучшение целостности (согласованности) данных.**
            *   **Легче изменять и обновлять данные.**
            *   **Независимость данных от программ** (можно менять структуру БД без переделки всех программ).
            *   **Лучший доступ к данным и информации.**
            *   **Стандартизация доступа к данным.**
            *   **Улучшенная защита данных.**
            *   **Эффективное использование ресурсов.**
        *   **Недостатки (Слайд 26):**
            *   **Большая сложность:** СУБД – сложные системы, их настройка требует знаний.
            *   **Сложнее восстанавливать после сбоя:** Если "падает" СУБД, может "лечь" вся система (в файловом подходе сбой одного файла затрагивал только одно приложение).
            *   **Дороже:** СУБД и квалифицированные специалисты могут стоить дороже.
    *   **Сравнительная таблица (Слайд 27):**
        *   **Хранение:** Файловый - ленты/файлы; БД - устройства прямого доступа (DASD).
        *   **Данные:** Файловый - последовательные файлы без связей; БД - таблицы со связями.
        *   **Организация:** Файловый - много дублей; БД - дубли минимизированы.
        *   **Обновления:** Файловый - много обновлений в разных файлах; БД - одно или несколько обновлений для поля.
        *   **Обработка:** Файловый - медленные запросы; БД - быстрые запросы.

# "Lesson 2 part 2: Data Management

Эта часть углубляется в проектирование баз данных, особенно реляционных, и затрагивает современные концепции вроде Big Data.

1.  **Нормализация (Normalization) (Слайд 2):**
    *   **Что это?** Процесс "приведения в порядок" структуры базы данных, чтобы она была эффективной и без проблем. Это как уборка и раскладывание вещей по полочкам.
    *   **Цели:**
        *   **Уменьшить избыточность данных** (чтобы одно и то же не хранилось много раз).
        *   **Избежать "неудобных" связей "многие-ко-многим"** (их разбивают на более простые).
        *   **Повысить стабильность и гибкость** базы данных.
    *   *Проще говоря:* Делаем так, чтобы данные хранились "умно", без лишних повторов и сложностей.

2.  **Правила ссылочной целостности (Referential Integrity Rules) (Слайд 2):**
    *   **Что это?** Это правила, которые следят за тем, чтобы связи между таблицами были корректными. Особенно важно, когда есть **внешние ключи**.
    *   **Пример:** Если в таблице "Заказы" есть поле "Код клиента" (внешний ключ), которое ссылается на таблицу "Клиенты", то нельзя добавить в "Заказы" запись с несуществующим кодом клиента. Сначала такой клиент должен появиться в таблице "Клиенты". Также нельзя удалить клиента из таблицы "Клиенты", если на него есть ссылки в "Заказах" (или нужно настроить, что произойдет с этими заказами).
    *   *Проще говоря:* Гарантия того, что все ссылки между таблицами ведут на реально существующие данные.

3.  **Столбцы (Columns) и Строки (Rows) в таблицах (Слайды 3, 4):**
    *   **Столбец (Column) или Поле (Field):** Это категория информации в таблице (например, "Имя", "Цена", "Дата рождения"). Определяет, какой атрибут сущности хранится.
    *   **Строка (Row) или Запись (Record):** Это набор данных обо одном экземпляре сущности (например, вся информация об одном клиенте или одном товаре).

4.  **Типы связей между таблицами (Table Relationships) (Слайды 5, 8):**
    Как записи в одной таблице могут быть связаны с записями в другой.
    *   **Один-к-одному (One-to-one):** Одной записи в таблице А соответствует только одна запись в таблице Б, и наоборот. (Редко используется, часто такие данные можно объединить в одну таблицу).
        *   *Пример:* "Сотрудник" и "Рабочее место" (если у каждого сотрудника строго одно место, и каждое место занято строго одним сотрудником).
    *   **Один-ко-многим (One-to-many):** Одной записи в таблице А может соответствовать много записей в таблице Б, но каждой записи в таблице Б соответствует только одна запись в таблице А. Это самый частый тип связи.
        *   *Пример:* Один "Клиент" (таблица А) может сделать много "Заказов" (таблица Б). Но каждый "Заказ" принадлежит только одному "Клиенту".
    *   **Многие-ко-многим (Many-to-many):** Одной записи в таблице А может соответствовать много записей в таблице Б, и одной записи в таблице Б может соответствовать много записей в таблице А. Такие связи обычно реализуются через третью, связующую таблицу (junction table) с двумя связями "один-ко-многим".
        *   *Пример:* "Студент" и "Курс". Один студент может изучать много курсов, и один курс могут изучать много студентов. Для этого создают таблицу "Запись_на_курс", где каждая строка связывает одного студента с одним курсом.

5.  **Проектирование баз данных – Модель данных (Data Model) (Слайды 6, 7, 9, 17, 18):**
    *   **Модель данных (Data Model):** Это "карта" или схема, которая показывает, какие сущности (таблицы) есть в базе данных и как они связаны между собой. Помогает проектировщикам правильно построить БД, улучшить качество и снизить затраты.
    *   **Диаграмма "сущность-связь" (Entity Relationship Diagram - ERD):** Графический инструмент для изображения сущностей и их связей. Очень полезна при разработке сложных БД. (Слайд 9, 17, 18)
        *   На диаграмме сущности обычно рисуют как прямоугольники, а связи – как линии между ними, с указанием типа связи (один-к-одному, один-ко-многим и т.д.).

6.  **Реляционная База Данных (Relational Database) (Слайд 10):**
    *   Это **самый распространенный тип** баз данных.
    *   Данные организованы в виде набора **таблиц** (которые называются "отношениями", отсюда и "реляционная").
    *   Таблицы **связаны между собой** через общие столбцы (ключи).
    *   Проектируются так, чтобы **минимизировать повторение данных**.

7.  **Реляционная Модель (The Relational Model) (Слайды 11-14):**
    *   Представляет данные в виде "трехмерной" структуры:
        *   **Строки (Rows):** Первое измерение (конкретные записи).
        *   **Столбцы (Columns):** Второе измерение (атрибуты).
        *   **Связи (Relationships):** Третье измерение (как строки из разных таблиц связаны). Это делает модель мощной, т.к. любая строка может быть связана с любой другой.
    *   Слайды 12, 13, 14 показывают примеры, как таблицы связываются для ответа на запросы (например, найти имя менеджера проекта, используя таблицы "Проект", "Отдел", "Менеджер").

8.  **Аспекты моделирования данных (Слайд 15):**
    При создании БД нужно продумать:
    *   **Содержимое (Content):** Какие данные собирать и сколько это стоит?
    *   **Доступ (Access):** Какие данные, каким пользователям и когда предоставлять?
    *   **Логическая структура (Logical structure):** Как данные должны быть организованы, чтобы быть понятными пользователю?
    *   **Физическая организация (Physical organization):** Где данные будут физически храниться?

9.  **Логическое и Физическое проектирование (Logical and Physical Design) (Слайд 16):**
    *   **Логическое проектирование:** Абстрактная модель структуры данных, отвечающая информационным потребностям организации (ЧТО храним и КАК это связано).
    *   **Физическое проектирование:** Начинается с логического дизайна и уточняет его с учетом производительности и затрат (КАК ИМЕННО это будет храниться на дисках, какие индексы и т.д.).
    *   **Планируемая избыточность данных (Planned data redundancy):** Иногда специально допускают небольшое дублирование данных, чтобы ускорить выполнение запросов или отчетов (это компромисс).

10. **Модель REA (Resources, Events, Agents - Ресурсы, События, Агенты) (Слайды 19, 20, 21):**
    *   Модель данных, где сущности представляют конкретные бизнес-**Ресурсы** (товары, деньги), **События** (транзакции, влияющие на ресурсы) и **Агентов** (люди или компании, участвующие в событиях).
    *   Полезна для проектирования БД для бизнес-систем, особенно бухгалтерских.
    *   Отличается от традиционных бухгалтерских терминов (актив, пассив), так как напрямую моделирует реальные бизнес-объекты.

11. **Схема (Schema) – Представление для пользователя (Слайды 22-25):**
    *   **Схема:** Это как "чертеж" или описание всей базы данных, показывающий структуру таблиц и связи между ними.
    *   **Три уровня схемы (представления данных):**
        1.  **Внешний уровень (External Level Subschema / End user view):** Как базу данных видит конкретный пользователь или приложение. Каждому пользователю могут быть доступны только нужные ему данные и в удобном ему виде. (Это подмножество общей схемы).
        2.  **Концептуальный уровень (Conceptual Schema / Design view):** Общее описание всей базы данных – какие данные хранятся и какие между ними связи. Это взгляд разработчика.
        3.  **Внутренний уровень (Internal Level Schema / Data dictionary view):** Описывает, как данные физически хранятся на диске (структуры файлов, индексы и т.д.). Это взгляд СУБД.

12. **СУБД (DBMS), Система Приложений БД (DB Application System), УИС (MIS) – как они связаны (Слайды 26, 27):**
    Это повторение и обобщение:
    *   Данные хранятся в **реляционной БД**.
    *   БД управляется **СУБД**.
    *   **Система приложений БД** = БД + СУБД + прикладные программы.
    *   **УИС (MIS)** = Система приложений БД + железо, другой софт, сети, люди, процедуры (т.е. вся информационная система).

13. **Использование MS Access как СУБД (Слайды 28, 29):**
    *   MS Access – это пример СУБД (и инструмент разработки ИС).
    *   Перед созданием БД в Access (или любой другой СУБД) **сначала нужно сделать дизайн** (схему таблиц, ключи, связи).
    *   Затем определяются интерфейсы (экраны) для работы с данными.

14. **Компоненты и Функции СУБД (Слайды 30-32):**
    *   **Словарь данных (Data Dictionary):** (Слайд 31) "Паспорт" базы данных. Описывает каждое поле (имя, тип данных, допустимые значения, ключ или нет, кто может обновлять). Помогает вводить данные и enforcing business rules (например, не давать вводить буквы в поле "возраст").
    *   **Язык запросов (Query Language):** (Слайд 30, 34, 35) Инструмент для извлечения данных из БД по определенным критериям.
        *   **SQL (Structured Query Language):** Стандартный язык для работы с реляционными БД. Позволяет выбирать, вставлять, обновлять, удалять данные.
            *   *Пример:* `SELECT Имя, Фамилия FROM Студенты WHERE Город = 'Москва'`
        *   **QBE (Query by Example):** Визуальный способ создания запросов, когда пользователь заполняет шаблон или перетаскивает поля.
    *   **Основные функции СУБД (Слайд 32):**
        *   Управление словарем данных.
        *   Преобразование и представление данных.
        *   Управление безопасностью.
        *   Резервное копирование и восстановление.
        *   Управление целостностью данных.
        *   Языки доступа к БД и интерфейсы для приложений.
        *   Интерфейсы связи.
        *   Управление транзакциями (чтобы операции выполнялись либо полностью, либо никак).
        *   Ввод данных, выполнение запросов и т.д.

15. **Ввод данных, Запросы, Отчеты в СУБД (Слайды 33-41):**
    *   **Ввод данных (Data Entry):** (Слайд 33) Данные вводятся через формы, которые часто соответствуют бумажным документам.
    *   **Запросы (Queries):** (Слайд 34-37) Способ извлечь нужную информацию. SQL – основной язык.
    *   **Отчеты (Reports):** (Слайд 38-41) Функция БД для извлечения и форматирования информации для печати или отображения. Генераторы отчетов используют SQL и шаблоны.

16. **Практические шаги по созданию Базы Данных (Слайды 42-58):**
    Это пошаговый процесс:
    1.  **Анализ требований (Data Requirements Analysis):** (Слайды 43-46) Поговорить с будущими пользователями системы. Узнать, какие им нужны отчеты и какая информация (какие поля данных). Все это записать.
    2.  **Определение элементов данных:** (Слайд 48) Составить список всех необходимых полей (атрибутов).
    3.  **Продумать хранение:** (Слайд 47) Просто хранить символы или поля недостаточно. Их нужно группировать в записи, а записи – в файлы/таблицы.
    4.  **Первоначальное внесение данных (для примера):** (Слайд 49) Попробовать заполнить одну большую таблицу всеми данными.
    5.  **Выявление неэффективности:** (Слайды 50, 51) Увидеть, что данные повторяются, хранение неэффективно. **Решение: Нормализовать базу данных!** (Разбить на несколько таблиц).
    6.  **Определение связей между данными:** (Слайд 52) Сгруппировать связанные поля в таблицы. Использовать логику. Определить, какие поля будут связывать таблицы (ключи). Создать модель данных (ERD).
    7.  **Пример нормализации и ERD:** (Слайды 53, 54) На примере БД "Школа" показано, как одна большая таблица разбивается на "Студент" и "Оценки", связанные номером студента.
    8.  **Определение ключей:** (Слайды 55-58) Для каждой таблицы определить первичные ключи. Определить внешние ключи для связей. Могут быть составные и вторичные ключи.

17. **Отчетность по данным (Data Reporting) (Слайды 59-62):**
    После того как БД создана и наполнена, пользователи могут получать нужную им информацию в виде отчетов и результатов запросов, основанных на их первоначальных требованиях.

18. **Большие Данные (Big Data) (Слайды 63-67):**
    *   **Что это?** Огромные и сложные наборы данных, с которыми традиционные инструменты управления данными не справляются.
    *   **Три "V" характеристики (Слайд 63):**
        *   **Volume (Объем):** Очень много данных (зеттабайты).
        *   **Velocity (Скорость):** Данные поступают очень быстро (триллионы бит в секунду).
        *   **Variety (Разнообразие):** Данные могут быть структурированными (таблицы) и неструктурированными (текст, видео, аудио, соцсети).
    *   **Источники (Слайд 64):** Документы, email, архивы, соцсети, данные с датчиков, логи машин, веб-сайты и т.д.
    *   **Генераторы (Слайд 65):** Большой адронный коллайдер, авиадвигатели, мобильные телефоны, YouTube, Facebook, Twitter.
    *   **Использование (Слайд 66):** Мониторинг соцсетей, анализ медицинских данных, поведение потребителей, целевой маркетинг.
    *   **Проблемы (Слайд 67):** Как выбрать, что хранить? Где и как хранить? Как найти ценные "самородки" данных? Как извлечь пользу? Как защитить?

19. **Управление данными и Руководство данными (Data Management and Governance) (Слайд 68):**
    *   **Управление данными (Data Management):** Цель – обеспечить, чтобы данные были доступными, надежными и своевременными для пользователей.
    *   **Руководство данными (Data Governance):** Определяет роли, обязанности и процессы, чтобы данным можно было доверять и использовать их во всей организации.
        *   **Распорядитель данных (Data Steward):** Человек, ответственный за управление критически важными элементами данных.

20. **Хранилища данных (Data Warehouses), Витрины данных (Data Marts) и Интеллектуальный анализ данных (Data Mining) (Слайды 69, 70, 73, 74):**
    *   **Хранилище данных (Data Warehouse - DW):** Большая база данных, которая собирает информацию из множества источников по всему предприятию (например, из операционных баз данных отделов). Данные извлекаются, преобразуются и загружаются в хранилище. Используется для бизнес-анализа.
    *   **Витрина данных (Data Mart):** Подмножество хранилища данных, предназначенное для конкретного отдела или бизнес-задачи (например, витрина данных для отдела маркетинга). Может содержать более детализированные данные по своей теме.
    *   **Интеллектуальный анализ данных (Data Mining):** Инструмент для автоматического обнаружения скрытых закономерностей, тенденций и связей в больших объемах данных (обычно в хранилищах данных).
        *   *Схема (Слайд 69, 73):* Операционные БД отделов -> (Extract) -> Хранилище Данных -> (Extract) -> Витрины Данных -> Анализ (руководители, отделы).

21. **Бизнес-аналитика (Business Intelligence - BI) (Слайды 71, 72):**
    *   **Что это?** Широкий спектр технологий и приложений, которые помогают организации преобразовывать (в основном структурированные) данные из информационных систем для проведения анализа, генерации информации и улучшения принятия решений.
        *   *Процесс:* Данные -> Информация -> Аналитика (Intelligence) -> Инновации -> Эффективность бизнеса.
    *   **Технологии BI включают:**
        *   **Data Mining** (поиск закономерностей "снизу вверх", на основе имеющихся данных).
        *   **Предиктивная аналитика (Predictive Analytics):** (Слайды 72, 74) Форма Data Mining, которая использует исторические данные и предположения о будущем для прогнозирования результатов (например, будущие продажи, вероятность дефолта клиента).
            *   *Пример:* Превращение случайных покупателей в постоянных, поиск новых сегментов рынка.

22. **Применения Data Mining (Слайд 75):**
    *   Брендинг и позиционирование продуктов.
    *   Прогнозирование оттока клиентов (Customer Churn).
    *   Прямой маркетинг (выявление тех, кто скорее всего откликнется).
    *   Обнаружение мошенничества (Fraud detection).
    *   Сегментация рынка.
    *   Анализ трендов (продажи, расходы, акции).

# Lesson 3: Management Information Systems Frameworks

1.  **Природа управленческой работы (Слайды 3-7):**
    *   **Менеджмент (Управление):** Это процесс направления задач и ресурсов для достижения целей организации.
    *   **Функции менеджмента:** Планирование, организация, руководство, мотивация, **контроль**.
    *   **Планирование:** Происходит на разных уровнях:
        *   **Долгосрочная миссия и видение** (куда идем в целом).
        *   **Стратегические цели** (крупные цели на несколько лет).
        *   **Тактические задачи** (конкретные шаги для достижения стратегических целей).
    *   **Важные виды планирования:** Составление графиков, бюджетирование, распределение ресурсов (персонал, оборудование и т.д.). (Слайд 4)
    *   **Контроль:** Менеджеры сравнивают фактические результаты с планами и вносят коррективы. (Слайд 5)
        *   *Примеры:* Отслеживание ресурсов проекта, сроков поставки, сравнение расходов с бюджетом, анализ отчетов об отклонениях.
    *   **Принятие решений:** И планирование, и контроль требуют принятия решений. Чем выше уровень управления, тем:
        *   Менее рутинная работа.
        *   Больше вариантов выбора.
        *   Больше ответственности за принятые решения. (Слайд 6)
    *   **Эффективность vs. Результативность (Efficiency vs. Effectiveness) (Слайд 7):**
        *   **Эффективность (Efficiency):** Делать вещи *правильно* (с минимальными затратами ресурсов).
        *   **Результативность (Effectiveness):** Делать *правильные* вещи (достигать поставленных целей).
        *   *Пример:* Убить комара кувалдой – результативно (комар убит), но не эффективно (слишком много усилий и ресурсов).

2.  **Управление по отклонениям (Management by Exception) (Слайд 8):**
    *   Это подход, когда менеджеры обращают внимание и действуют только в тех ситуациях, где фактические результаты **значительно** отличаются от запланированных. Это помогает не отвлекаться на мелочи.
    *   *Пример:* Если расходы на телефонную связь превысили бюджет на 10.7% – это отклонение, требующее внимания.

3.  **Уровни принятия решений в организации (Организационная пирамида) (Слайды 9-13, 17, 82, 85):**
    В организации обычно выделяют три основных уровня управления, каждому из которых нужны свои типы информации и информационных систем:
    *   **Операционный уровень (Operational Level):**
        *   **Кто:** Мастера, бригадиры, рядовые сотрудники, клерки.
        *   **Задачи:** Ежедневные операции, рутинные задачи (обработка заказов, учет запасов, начисление зарплаты).
        *   **Решения:** **Операционные решения** – ежедневные, повторяющиеся, с низким риском, основаны на четких правилах и процедурах. (Слайд 12)
            *   *Пример:* Отказать в кредите клиенту с просроченной задолженностью.
        *   **Информационные потребности:** Детальная, точная, текущая информация о конкретных операциях.
        *   **Информационная система:** **Системы обработки транзакций (TPS - Transaction Processing Systems).** (Слайд 11, 20-43)
    *   **Тактический (Управленческий) уровень (Tactical/Managerial Level):**
        *   **Кто:** Менеджеры среднего звена, руководители отделов.
        *   **Задачи:** Реализация планов высшего руководства, контроль деятельности подразделений, бюджетирование, составление графиков.
        *   **Решения:** **Тактические решения** – реализация стратегий, разработка планов для подразделений, распределение ресурсов. Риск выше, чем на операционном уровне. (Слайд 12)
            *   *Пример:* Анализ продаж по товарным линиям в регионе, сравнение плановых и фактических расходов отдела.
        *   **Информационные потребности:** Обобщенная информация (сводки, итоги), сравнение с планами, анализ отклонений. Информация как внутренняя, так и частично внешняя.
        *   **Информационная система:** **Управленческие информационные системы (MIS - Management Information Systems).** (Слайд 11, 44-62)
    *   **Стратегический (Исполнительный) уровень (Strategic/Executive Level):**
        *   **Кто:** Высшее руководство (директора, президенты).
        *   **Задачи:** Долгосрочное планирование, определение миссии и целей организации, принятие решений о выходе на новые рынки, разработке новых продуктов.
        *   **Решения:** **Стратегические решения** – затрагивают всю организацию, связаны с серьезными изменениями, направлены на достижение долгосрочных целей. Высокий уровень неопределенности и риска. (Слайд 12)
            *   *Пример:* Использование модели для прогнозирования прибыльности нового продукта, открытие нового завода на новом рынке.
        *   **Информационные потребности:** Высокоагрегированная (сводная) информация, прогнозы, тренды, информация о конкурентах, рынке, экономике (много внешней информации).
        *   **Информационная система:** **Информационные системы для руководителей (EIS - Executive Information Systems) или Системы поддержки руководителей (ESS - Executive Support Systems).** (Слайд 11, 63-81)

4.  **Новые реалии в бизнесе и взгляд в будущее (Слайды 14-16):**
    Бизнес меняется, и ИТ играют в этом ключевую роль:
    *   Организации становятся **информационно-ориентированными** и **сетевыми** (вместо иерархических).
    *   Растет **внимание к клиентам** и их предпочтениям, переход от массового производства к **кастомизации** (индивидуализации продуктов/услуг), что возможно благодаря ИТ.
    *   **Обслуживание клиентов** становится критически важным.
    *   **Глобализация:** Всемирные коммуникации позволяют работать на глобальных рынках.
    *   **Реинжиниринг бизнес-процессов:** Радикальное переосмысление и изменение способов ведения деятельности с помощью ИТ.
    *   В будущем: больше **сетевых организаций специалистов**, работа потребует **высокоуровневого мышления и постоянного обучения**, критическое мышление и инновации будут ключевыми. ИТ останутся центральным элементом бизнеса.

5.  **Три типа организационных систем (Слайды 19, и далее подробно):**
    Это те самые TPS, MIS, EIS/ESS, рассмотренные в контексте уровней управления.
    *   **Системы Обработки Транзакций (TPS - Transaction Processing Systems) (Слайды 20-43):**
        *   **Уровень:** Операционный.
        *   **Кто использует:** Рядовые сотрудники, мастера (Foremen or Supervisor). (Слайд 21)
        *   **Что делают:** Автоматизируют рутинные и повторяющиеся операции и события. Собирают и обрабатывают детальные данные о повседневных бизнес-операциях (заказы, платежи, складской учет, зарплата и т.д.). (Слайды 22, 28)
        *   **Зачем:** Повышение эффективности организации. (Слайд 21)
        *   **Характеристики (Слайд 23):** Повторяемость, структурированные вход/выход данных, предсказуемость, акцент на прошлом (регистрация фактов), высокодетализированные данные, внутренние источники.
        *   **Транзакция (Слайд 24, 25):** Любое событие, которое что-то меняет (например, заказ клиента, оплата счета, поступление товара). Финансовая транзакция – обмен платежом. Компьютерная транзакция – обмен информацией и обновление БД.
        *   **Архитектура (Слайд 18, 26):** Ввод (событие) -> Процесс (программы TPS, запись, сортировка, обновление) -> Вывод (отчеты, данные для других систем).
        *   **Методы обработки транзакций (Слайд 27):**
            *   **Пакетная (Batch):** Транзакции накапливаются и обрабатываются все вместе, "пачкой" (например, обработка всех платежей в конце дня).
            *   **Онлайн (Real-time):** Транзакции обрабатываются немедленно, по мере их возникновения (например, бронирование билета онлайн).
        *   **Цели TPS (Слайд 29):** Собирать, обрабатывать, обновлять БД; обеспечивать точность и полноту; избегать мошенничества; своевременно выдавать отчеты; снижать трудозатраты; улучшать обслуживание клиентов; давать конкурентное преимущество.
        *   **Примеры TPS (Слайд 30):** Системы обработки заказов, бухгалтерские системы, системы закупок.
        *   **Цикл обработки транзакций (Слайд 31):** Сбор данных -> Редактирование (проверка) -> Коррекция ошибок -> Обработка (вычисления, сортировка) -> Хранение данных -> Создание документов/отчетов. (Слайды 32-37 подробно)
        *   **Примеры отчетов TPS (Слайды 40-43):** Детальные отчеты о продажах по категориям, трафике, конверсии.
    *   **Управленческие Информационные Системы (MIS - Management Information Systems) (Слайды 44-62):**
        *   **Уровень:** Тактический/Управленческий.
        *   **Кто использует:** Менеджеры среднего звена, функциональные менеджеры. (Слайд 45)
        *   **Что делают:** Автоматизируют мониторинг и контроль операционной деятельности. Предоставляют менеджерам регулярные отчеты об операциях для контроля, организации и планирования. Показывают фактические данные в сравнении с плановыми, измеряют прогресс. (Слайды 46, 48)
        *   **Зачем:** Повышение результативности организации. (Слайд 45)
        *   **Функции MIS (Слайд 47):** Предоставление отчетов в стандартных форматах, создание бумажных и электронных отчетов, возможность создания пользовательских отчетов.
        *   **Отличие от TPS (Слайд 50, 51):** TPS собирает детали, MIS их обобщает для менеджеров. MIS предоставляет информацию для контроля, а не для выполнения операционных задач. Информация в MIS менее детализирована, более сравнительная, структура менее жесткая, может формироваться не так регулярно (иногда ad-hoc).
        *   **Входные данные MIS (Слайд 55):** Данные из TPS, другие внутренние данные, запросы на информацию. Частично могут использоваться внешние источники (Слайд 57).
        *   **Выходные данные MIS (Слайды 52, 55, 58, 59, 71-73):**
            *   **Плановые отчеты (Scheduled Reports):** Регулярные (ежедневные, еженедельные, ежемесячные).
                *   **Отчеты по ключевым показателям (Key Indicator Reports):** Сводки для мониторинга производительности (например, месячный отчет о продажах). (Слайд 58)
                *   **Отчеты об исключениях (Exception Reports):** Подсвечивают ситуации, выходящие за рамки нормы (например, просроченные поставки). (Слайд 58, 72)
            *   **Отчеты по запросу (Ad Hoc Reports):** Незапланированные, создаются для решения конкретной проблемы. (Слайды 53, 59, 73)
            *   **Drill-Down (Детализация):** Возможность "провалиться" из сводного отчета к более детальной информации. (Слайды 53, 54)
        *   **Примеры отчетов MIS (Слайды 60-62):** Отчеты об отставании заказов, росте клиентской базы, дашборды с продажами по магазинам.
    *   **Системы Поддержки Руководителей (ESS - Executive Support Systems) / Информационные Системы для Руководителей (EIS - Executive Information Systems) (Слайды 63-81):**
        *   **Уровень:** Стратегический/Исполнительный.
        *   **Кто использует:** Высшее руководство. (Слайд 64)
        *   **Что делают:** Предоставляют обобщенные данные и прогнозы для долгосрочного стратегического планирования и принятия решений. Помогают определять видение, разрабатывать стратегию, контролировать ее выполнение, управлять кризисами. (Слайды 65, 66, 68)
        *   **Зачем:** Улучшение стратегии и планирования организации. (Слайд 64)
        *   **Характеристики:** Используют графические интерфейсы, показывают консолидированную информацию. Могут работать как с "мягкими" данными (Soft Data - новости, неструктурированные тексты), так и с "жесткими" (Hard Data - цифры, факты). (Слайд 68)
        *   **Входные данные EIS/ESS (Слайды 69, 70):** Внутренние (TPS, MIS, хранилища данных, витрины данных) и **значительная часть внешних данных** (клиенты, поставщики, конкуренты, Интернет, рыночная аналитика).
        *   **Выходные данные EIS/ESS (Слайды 69, 71-74, 77-81):**
            *   **Сводные отчеты (Summary Reports).**
            *   **Анализ трендов (Trend Analysis).**
            *   **Моделирование (Simulations).**
            *   **Drill-Down (Детализация).** (Слайд 74)
            *   **Дашборды (Dashboards):** (Слайды 77-80) Визуальное представление ключевых показателей эффективности (KPI) в легко интерпретируемом виде. Часто используются в ESS.
        *   **Ключевые показатели эффективности (KPI - Key Performance Indicators) (Слайды 75, 76):** Метрики, которые отслеживают прогресс в достижении организационных целей. Состоят из направления (увеличить/уменьшить), измеряемого показателя, целевого значения и временных рамок.
        *   **Примеры отчетов EIS (Слайд 81):** Дашборды с недельными продажами, запасами, эффективностью сотрудников, сравнением с прошлым годом.

6.  **Функциональные системы (Functional Area systems) (Упоминается в целях, слайд 2):**
    *   Это информационные системы, которые поддерживают конкретные функциональные области бизнеса (например, маркетинг, финансы, производство, управление персоналом). Они могут пересекать границы отделов и уровней управления. Хотя подробно не раскрыты, важно понимать, что ИС часто специализированы по функциям.

# Lesson 4 part 1: Functional Area Systems: Accounting Transaction Systems

1.  **Функциональные Информационные Системы (Слайды 2, 3):**
    *   **Что это?** Информационные системы, разработанные для поддержки конкретных **функциональных областей** бизнеса.
    *   **Примеры функциональных областей и их ИС (Слайд 2):**
        *   **Учет и Финансы:** Управление запасами, кредиторская задолженность, расходы, управление денежными средствами, расчет заработной платы. (Именно об этом данная лекция).
        *   **Управление персоналом (HR):** Набор и найм, обучение, управление льготами, увольнение, планирование рабочей силы.
        *   **Маркетинг:** Исследование рынка, разработка новых продуктов, продвижение и реклама, ценообразование, анализ продаж, размещение продукта.
        *   **Производство и Операции:** Управление запасами, отслеживание затрат и качества, планирование материалов и ресурсов, отслеживание обслуживания клиентов, отслеживание проблем клиентов, калькуляция работ, использование ресурсов.
    *   **Пирамида ИС по функциям (Слайд 3):** Показывает, что функциональные ИС (бухгалтерские, финансовые, HR, маркетинговые, операционные) являются основой для систем более высокого уровня (обработка заказов, финансовое прогнозирование), которые, в свою очередь, поддерживают стратегические системы (анализ коэффициентов, управление портфелем, кадровое прогнозирование и т.д.).

2.  **Бухгалтерские Информационные Системы (AIS - Accounting Information Systems) (Слайды 4, 5):**
    *   **Что делают AIS?** Отслеживают и документируют те аспекты бизнес-операций, которые связаны с:
        *   **Ответственностью за активы и обязательства** предприятия.
        *   **Определением результатов операций** (расчет прибыли/убытка).
        *   **Финансовой отчетностью.**
    *   **Основное требование:** Все финансовые транзакции должны отражаться в единой бухгалтерской системе, способной формировать как минимум два главных финансовых отчета: **Баланс (Balance Sheet)** и **Отчет о прибылях и убытках (Income Statement)**.
    *   **Бухгалтерский учет по своей сути:** Это процесс сбора, идентификации и хранения данных, а также сама информационная система.
    *   **AIS собирает, записывает, хранит и обрабатывает** бухгалтерские и другие данные для получения информации.
    *   **На какие вопросы помогает ответить AIS (Слайд 5):**
        *   Сколько и какой у нас долг?
        *   Продажи в этом периоде выше, чем в прошлом?
        *   Какие у нас активы?
        *   Какие были притоки и оттоки денежных средств?
        *   Получили ли мы прибыль в прошлом периоде?

3.  **Бизнес-процессы и Циклы транзакций (Transaction Cycles) (Слайд 6):**
    *   Многие бизнес-операции – это обмен "дал-получил" (give-get exchange). Например, отдаем деньги за товар, или платим зарплату сотрудникам за их труд.
    *   Чтобы правильно собирать данные и формировать информацию для принятия решений, компании группируют эти обмены "дал-получил" в **Циклы транзакций**.
    *   Системы обработки транзакций (TPS) фиксируют данные по мере выполнения этих бизнес-процессов.
    *   Циклы обычно обрабатывают несколько связанных транзакций многократно.

4.  **Основные Циклы Обработки Транзакций (Слайды 7, 8):**
    *   **Цикл доходов (Revenue Cycle):** Продажа товаров/услуг за наличные или с обещанием оплаты в будущем. *Дал: Товар/Услугу, Получил: Деньги/Обещание денег.*
    *   **Цикл расходов (Expenditure Cycle):** Покупка сырья, материалов или запасов для перепродажи в обмен на наличные или с обещанием заплатить в будущем. *Дал: Деньги/Обещание денег, Получил: Товар/Сырье.*
    *   **Цикл производства (Production or Conversion Cycle):** Превращение сырья в готовую продукцию. *Дал: Сырье + Труд, Получил: Готовую продукцию.*
    *   **Цикл управления персоналом/Расчета зарплаты (Human Resources/Payroll Cycle):** Найм, обучение, оплата труда, оценка, продвижение и увольнение сотрудников. *Дал: Деньги (зарплата), Получил: Труд.*
    *   **Цикл финансирования (Financing Cycle):** Продажа акций инвесторам, получение займов, выплата дивидендов и процентов по займам. *Дал (инвесторам/кредиторам): Дивиденды/Проценты, Получил: Деньги (инвестиции/займы).*
    *   **Центральное звено (Слайд 8):** Все эти циклы поставляют данные в **Главную книгу (General Ledger)**, которая используется для формирования отчетности для внутренних и внешних пользователей.

5.  **Бухгалтерские TPS: Циклы транзакций (Слайд 9):**
    *   Каждый цикл может быть реализован в ИС с помощью отдельных модулей.
    *   Не каждой организации нужны все модули.
    *   Каждый цикл включает множество процессов и операций.
    *   В конечном итоге все данные попадают в **Систему главной книги и отчетности (General Ledger and Reporting System)** для обновления главной книги и подготовки отчетов.

6.  **Системы Обработки Транзакций (TPS) для Бухгалтерии (Слайд 10):**
    *   Примеры ИС, поддерживающих бухгалтерские операции: ввод заказов, дебиторская задолженность, кредиторская задолженность, закупки и получение товаров, учет запасов, учет основных средств.

7.  **Цикл обработки транзакций (в целом) (Слайд 11):**
    *   Начинается со сбора данных о транзакции.
    *   Заканчивается выводом информации, такой как финансовые отчеты.

8.  **Два основных цикла бухгалтерских транзакций (Слайды 12, 13):**
    Лекция фокусируется на двух главных:
    *   **Цикл доходов (Revenue cycle):** Продажа товаров/услуг и сбор платежей.
    *   **Цикл расходов (Expenditure cycle):** Покупка товаров/услуг и оплата за них.
    Оба цикла поставляют данные в Систему главной книги и отчетности.

9.  **Базовая модель циклов бухгалтерских транзакций (Слайд 14):**
    Показывает, как циклы закупок (от поставщиков) и продаж (клиентам) взаимодействуют с банковскими операциями, финансовым учетом и формированием отчетности.

10. **Цикл Доходов (Продаж) – Revenue (Sales) Cycle (Слайды 15-33):**
    *   **Первичные документы и их функции (Слайд 15):**
        *   **Заказ на продажу (Sales order):** Принять заказ клиента.
        *   **Накладная на отгрузку (Delivery ticket):** Доставить или отгрузить заказ.
        *   **Уведомление о денежном переводе (Remittance advice):** Получить наличные.
        *   **Слип на инкассацию (Deposit slip):** Внести наличные поступления на счет.
        *   **Кредит-нота (Credit memo):** Скорректировать счета клиентов (например, при возврате товара).
    *   **Основные операции цикла продаж (Слайды 16, 17, 18):**
        1.  Выставление счетов (Create Invoices).
        2.  Получение платежей от клиентов (Receive Customer Payments).
        3.  Внесение депозитов (Make Deposits).
    *   **Диаграммы бизнес-процессов (Слайды 19, 21, 23, 24):**
        *   **Swim Lane Diagram (Диаграмма с дорожками):** Показывает, какие отделы/роли (например, Клиент, Бухгалтерия, Банк) выполняют какие действия в процессе.
        *   **BPMN (Business Process Model and Notation):** Стандартная графическая нотация для моделирования бизнес-процессов. Показывает последовательность действий, решения (шлюзы), потоки данных, исполнителей.
    *   **Пример процесса обработки заказа клиента (Espresso Coffee) (Слайды 20-28):**
        1.  Клиент делает онлайн-заказ.
        2.  Компания обрабатывает заказ: проверяет клиента, наличие товара, кредитоспособность (через компанию кредитных карт).
        3.  Разрешение на отгрузку отправляется на склад.
        4.  Склад собирает товар, отгружает клиенту, уведомляет бухгалтерию.
        5.  Бухгалтерия генерирует счет и отправляет клиенту. Обновляются таблицы в БД.
    *   **Чтение BPMN-диаграммы (Слайд 22):** План анализа диаграммы: найти начало, действия, шлюзы (решения), объекты данных, хранилища данных, потоки данных, конец. Затем создать организатор карты процесса, описать процесс текстом, связать текст с картой, создать таблицы БД.
    *   **Разбор процесса на примере BPMN (Слайды 23-28):** Пошаговый анализ BPMN-диаграммы для процесса "Создание счетов": высокоуровневая карта, детальная карта, организатор действий, текстовое описание, аннотированное описание, связь с таблицами БД.
    *   **Процесс получения платежей от клиентов (Слайды 29-33):** Аналогично разбирается на диаграммах (Flowchart, BPMN), с организатором действий и текстовым описанием.

11. **Цикл Расходов (Закупок) – Expenditure (Purchasing) Cycle (Слайды 34-46):**
    *   **Первичные документы и их функции (Слайд 35):**
        *   **Заявка на закупку (Purchase requisition):** Запросить товары.
        *   **Заказ на закупку (Purchase order):** Заказать товары.
        *   **Отчет о приемке (Receiving report):** Получить товары.
        *   **Чек (Check):** Оплатить товары.
    *   **Основные операции цикла закупок (Слайды 34, 36):**
        1.  Создание заказов на закупку (Create purchase orders).
        2.  Получение товаров (Receive items).
        3.  Ввод счетов от поставщиков (Enter bills).
        4.  Оплата счетов (Pay bills).
    *   **Пример процесса обработки заказа на закупку (Espresso Coffee) (Слайды 37-46):**
        1.  Компания создает заказ на закупку для поставщика.
        2.  Отгрузка подтверждается и отправляется на склад компании, где товар принимается на учет.
        3.  Склад генерирует отчет о приемке и обновляет данные в бухгалтерии.
        4.  Бухгалтерия получает счет от поставщика, сверяет его с отчетом о приемке и обрабатывает счет.
    *   **Разбор подпроцессов на BPMN (Слайды 38-46):** "Создание заказов на закупку", "Получение товаров", "Обработка счета (Process Bill)", "Оплата счета (Pay Bill)" – представлены в виде диаграмм и таблиц-организаторов.

# Lesson 5: Functional Area Systems: Finance, Human Resources, Marketing and Sales, Production/Operations

#### 1. Финансовые Информационные Системы (Financial Information Systems) (Слайды 3-16)

*   **Цель Финансов (Слайд 4):** Предоставлять внутренним и внешним пользователям доступ к информации об акциях, инвестициях и капитальных затратах. Основные задачи финансового отдела:
    *   Привлекать и управлять денежными средствами.
    *   Привлекать средства через акции или займы.
    *   Разумно инвестировать денежные средства.
    *   Поддерживать ликвидность (способность выполнять краткосрочные обязательства).
    *   Анализировать значительный объем финансовой информации.
*   **Структура Финансовых ИС (Слайд 5):** Показывает, как операционные системы (Главная книга, Обработка заказов, Кредиторская/Дебиторская задолженность, Зарплата, Учет запасов) поставляют данные для тактических систем (Бюджетирование, Управление инвестициями, Капитальное бюджетирование, Управление затратами, Налоговый менеджмент, Аудит, Финансовое планирование), которые в свою очередь поддерживают стратегические системы (Стратегическое планирование, Анализ коэффициентов и финансового здоровья, Анализ слияний и поглощений). Все это взаимодействует с внутренней и внешней средой.
*   **Типы Финансовых ИС (Слайд 6):**
    *   **Тактические Финансовые Системы:**
        *   Системы бюджетирования.
        *   Системы управления денежными средствами.
        *   Системы капитального бюджетирования.
        *   Системы управления инвестициями.
    *   **Стратегические Финансовые Системы:**
        *   Системы анализа финансового состояния.
        *   Системы долгосрочного прогнозирования.
        *   Системы корпоративного планирования.

*   **Тактические Системы в Финансах (Слайды 7-11):**
    *   **Системы Бюджетирования (Budgeting Systems) (Слайды 7, 8):**
        *   Планируют доходы и расходы по статьям.
        *   Могут обобщать данные различными способами.
        *   Сравнивают текущий бюджет с прошлогодним.
        *   Сравнивают фактические расходы с бюджетными (данные из Главной книги) – это называется **анализ отклонений (variances analysis)**.
        *   Полезны системы типа электронных таблиц (с формулами).
        *   Можно использовать бюджет прошлого года как основу для планирования на несколько лет вперед.
    *   **Системы Управления Денежными Средствами (Cash Management Systems) (Слайд 9):**
        *   Обеспечивают наличие достаточного количества денег для текущих нужд (операции, покупка долгосрочных активов).
        *   Важный выход – **отчет о движении денежных средств (cash flow report)**.
        *   Могут прогнозировать движение денежных средств (например, на 6 месяцев). Денежный поток = общие поступления – общие платежи.
    *   **Системы Капитального Бюджетирования (Capital Budgeting Systems) (Слайд 10):**
        *   Помогают планировать приобретение (или выбытие) **крупных основных средств** (например, оборудование, здания), которые будут использоваться много лет.
        *   Учитывают входящие и исходящие денежные потоки за весь срок службы актива.
        *   Используют допущения и учитывают **временную стоимость денег (чистая приведенная стоимость - net present value)**.
        *   *Пример:* Помочь решить, арендовать или купить новый принтер.
    *   **Системы Управления Инвестициями (Investment Management Systems) (Слайд 11):**
        *   Управляют инвестициями организации в акции, облигации и другие ценные бумаги.
        *   Онлайн-базы данных предоставляют актуальные цены на акции и облигации.
        *   *Пример:* Система может рассчитать прибыль или убыток по инвестиционному портфелю компании, если бы он был продан сейчас.

*   **Стратегические Системы в Финансах (Слайды 12-16):**
    *   **Характеристики (Слайд 12):**
        *   Поддерживают высшее руководство.
        *   Связаны с целями организации.
        *   Используют внутренние данные (из AIS) и **внешние данные** (экономические, социальные, демографические из онлайн-баз).
    *   **Системы Анализа Финансового Состояния (Financial Condition Analysis Systems) (Слайд 13):**
        *   Предоставляют глубокий анализ финансовых отчетов и данных (например, расчет коэффициентов: текущей ликвидности, соотношения долга к собственному капиталу).
        *   Онлайн-базы данных позволяют анализировать финансовое состояние конкурентов, поставщиков, покупателей.
        *   **Примеры финансовых коэффициентов (Слайд 14):**
            *   **Коэффициент денежной наличности (Cash Ratio):** (Денежные средства + Эквиваленты + Инвестиции) / Текущие обязательства. Показывает способность покрыть текущие обязательства.
            *   **Коэффициент текущей ликвидности (Current Ratio):** Текущие активы / Текущие обязательства. Индикатор способности правительства (или компании) выполнять краткосрочные обязательства.
            *   **Операционный коэффициент (Operating Ratio):** Общие доходы / Общие расходы. Если > 1.00 – профицит бюджета, < 1.00 – дефицит. Важен для краткосрочного финансового состояния.
            *   **Коэффициент соотношения заемного и собственного капитала (Debt/Equity Ratio):** Общие обязательства / Собственный капитал акционеров. Показывает, сколько заемных средств компания использует для финансирования своих активов относительно собственного капитала.
    *   **Системы Долгосрочного Прогнозирования (Long Range Forecasting Systems) (Слайд 15):**
        *   Используют как внутренние, так и внешние данные.
        *   Применяют статистические методы.
        *   Могут использовать графику.
        *   Анализируют тренды.
    *   **Системы Корпоративного Планирования (Corporate Planning Systems) (Слайд 16):**
        *   Используют данные из прошлого (как и системы прогнозирования).
        *   Более сложные; содержат **имитационные модели (simulation models)** различных аспектов бизнеса.
        *   Модели должны основываться на достоверных допущениях.

#### 2. Информационные Системы Управления Персоналом (Human Resource Information Systems - HRIS или HRMS) (Слайды 17-28)

*   **Обзор HR-систем (Слайд 17, 18):** Поддерживают управление, контроль и аудит человеческих ресурсов. Примеры: наем, обучение, управление льготами, увольнение, планирование рабочей силы.
*   **HRMS (Системы Управления Человеческими Ресурсами) (Слайд 19):**
    *   Комбинация систем и процессов, связывающих управление персоналом и ИТ через программное и аппаратное обеспечение.
    *   **Функции:** Управление зарплатой, рекрутмент и адаптация новых сотрудников (onboarding), сбор, хранение и доступ к информации о сотрудниках, учет посещаемости и прогулов, оценка производительности, администрирование льгот, управление обучением, самообслуживание сотрудников, планирование графиков работы, аналитика и принятие обоснованных решений.
*   **Операционные Системы в HR (Слайды 20-23):**
    *   **Расчет заработной платы (Payroll):** Исторически первая автоматизированная функция HR, но часто рассматривается как часть AIS (цикла расходов). (Слайд 20, 21)
        *   *Процесс (Слайд 21):* Поддержание данных о зарплате -> Валидация данных о времени и посещаемости -> Подготовка расчета зарплаты (пакетная обработка) -> Выплата зарплаты (чеки, прямой перевод) -> Расчет и перечисление налогов и отчислений.
    *   **Информационные системы о сотрудниках (Employee information systems) (Слайд 20):**
        *   Хранят информацию о каждом сотруднике: личные данные, образование, опыт, история работы в организации, предпочтения по месту работы.
        *   Могут содержать **инвентаризацию навыков (skills inventory)**: опыт, предпочтения, результаты тестов, интересы, особые навыки. Это может использоваться для подбора кандидатов на внутренние вакансии.
    *   **Системы учета посещаемости (Attendance recording systems) (Слайд 22):**
        *   Могут использовать "негативную отчетность" (отмечаются только отсутствующие).
        *   Учитывают сверхурочные и т.д.
    *   **Системы планирования графиков работы сотрудников (Employee Scheduling Systems) (Слайд 22):**
        *   Могут быть сложными при сменной работе (например, у медсестер).
        *   Должны учитывать профсоюзные регуляции.
    *   **Системы управления производительностью (Performance Management Systems) (Слайд 23):**
        *   Собирают и хранят текстовые данные (например, комментарии руководителя).
        *   Данные об оценке могут вводиться через специальные экраны.
        *   Требуют тщательного документирования производительности и методов ее измерения (важно для разбирательств).
        *   Могут иметь тактические компоненты: выявление руководителей, ставящих много плохих оценок; выявление источников неквалифицированной рабочей силы.
        *   Могут включать показатели производительности (например, количество обработанных заказов в день).
*   **Тактические Системы в HR (Слайды 24, 25):**
    *   **Системы контроля должностей (Position control systems) (Слайд 24):**
        *   Хранят данные о каждой должности в организации (например, содержание задач).
        *   Полезны для редизайна должностей.
        *   Помогают определить, какие должности требуют ввода данных, какие – статистического анализа.
    *   **Системы рекрутинга (Recruiting systems) (Слайд 24):**
        *   Предоставляют список планируемых выходов на пенсию.
        *   Перечисляют навыки и предпочтения текущих сотрудников (для внутреннего найма).
        *   Анализируют уровень текучести кадров по различным категориям сотрудников.
    *   **Системы компенсаций и льгот (Compensation and benefit systems) (Слайд 25):**
        *   Поддерживают льготы в стиле "кафетерия" (сотрудники выбирают из набора).
        *   Требуют значительного хранения данных.
        *   **Тактические вопросы:** Насколько увеличить компенсационные планы для привлечения высококвалифицированных сотрудников? Какие льготы выбирают разные категории сотрудников?
        *   Могут быть доступны через интранет организации.
*   **Стратегические Системы в HR (Слайд 26):**
    *   **Долгосрочное планирование рабочей силы:**
        *   Каковы потребности HR для выполнения стратегического плана организации на следующие 5-10 лет?
        *   Определение текущих потребностей в HR.
        *   Выявление критически важных должностей.
        *   Определение текущих и будущих "пробелов" в рабочей силе.
        *   Выявление факторов, влияющих на состав рабочей силы.
        *   Прогнозирование спроса и предложения необходимой рабочей силы.
        *   Формулирование планов.
    *   **Системы поддержки переговоров с профсоюзами (Labour negotiation support systems):**
        *   Должны быть своевременными и иметь возможность формировать отчеты по запросу (ad hoc).
        *   Помогают на переговорах с профсоюзами.
*   **Программное обеспечение HRMS (Слайд 27):**
    *   HR-системы хранят гораздо больше текстовых данных, чем другие функциональные системы.
    *   Существуют специализированные HR-системы.
    *   Использование HRMS не так распространено в малом и среднем бизнесе.
    *   **Пример интерфейса HRMS (Слайд 28).**

#### 3. Информационные Системы Маркетинга и Продаж (Marketing and Sales Information Systems) (Слайды 29-39)

*   **Обзор (Слайд 29, 30):** Поддерживают управление разработкой новых продуктов, дистрибуцией, ценообразованием, эффективностью продвижения и прогнозированием продаж.
*   **Функция Маркетинга (Слайд 31):**
    *   **Маркетинг:** Процесс планирования и выполнения концепции, ценообразования, продвижения, продаж и распространения идей, товаров и/или услуг для создания обменов, удовлетворяющих индивидуальные (клиентские) и организационные (бизнес) цели.
    *   **Маркетинговая Информационная Система (MIS, здесь в значении Marketing IS):** Набор процедур и методов для регулярного, планового сбора, анализа и представления информации для принятия маркетинговых решений.
*   **Необходимость координации систем (Слайд 32):** Для успеха маркетинговые системы должны быть скоординированы с другими системами организации, такими как: системы ввода заказов, производственные системы, системы управления запасами, системы управления кредитами.
*   **Операционные Системы в Маркетинге (Слайды 33-35):**
    *   **Системы управления контактами с клиентами (Customer contact management systems) (Слайд 33):**
        *   Ключевые для отслеживания и управления взаимодействиями с клиентами.
        *   Предоставляют информацию о прошлых контактах (даты, темы обсуждения, последующие действия).
        *   **Выход:** Отчет о звонке/контакте (Call report) с деталями взаимодействия и результатами.
        *   **Ключевые показатели:** Количество звонков, сделанных продавцом; количество и сумма продаж, сделанных продавцом. Помогают улучшать стратегии продаж и отслеживать производительность.
    *   **Системы Автоматизации Работы Отдела Продаж (Sales Force Automation - SFA) (Слайд 34):**
        *   Предназначены для оптимизации процесса продаж.
        *   **Управление процессом/активностью продаж:** Помогают организовывать задачи, встречи, последующие действия, чтобы не пропустить ни одного шага в последовательности продаж (генерация возможности -> распределение лида -> контакт с потенциальным клиентом -> квалификация -> определение решения -> размещение заказа).
        *   **Повышение продуктивности:** Помогают сосредоточить усилия на наиболее прибыльных клиентах.
    *   **Системы телемаркетинга (Telemarketing systems) (Слайд 35):**
        *   Идентифицируют клиентов и автоматически их обзванивают.
        *   Используют электронные телефонные справочники.
        *   Позволяют делать заметки о звонках.
        *   В системе на базе LAN могут одновременно работать много телемаркетологов.
    *   **Системы прямой почтовой рекламы (Direct mail advertising systems) (Слайд 35):** Создают почтовые этикетки.
    *   **Системы отслеживания и маршрутизации доставки (Delivery tracking and routing systems) (Слайд 35):** Помогают планировать оптимальные маршруты доставки.
*   **Тактические Системы в Маркетинге (Слайд 36):**
    *   **Цель тактических маркетологов:** Достижение целей продаж, установленных высшим руководством маркетинга.
    *   **Тактические решения:** Как формировать территории продаж? Как распределять продавцов по территориям? Какие продукты предлагать каким клиентам?
*   **Стратегические Системы в Маркетинге (Слайды 37, 38):**
    *   Могут содержать как стратегические, так и тактические элементы.
    *   **Системы прогнозирования продаж (Sales forecasting systems) (Слайд 37):**
        *   Прогнозируют продажи для всей отрасли, для всей организации, для каждого продукта, для сегментов рынка.
        *   Используют сложные **статистические модели** и могут давать значительный графический вывод.
    *   **Системы маркетинговых исследований (Market research systems) (Слайд 38):**
        *   Обрабатывают результаты опросов и интервью.
        *   Предоставляют анализ статистической значимости.
        *   Используют значительный объем **внешних данных**.
*   **Связанная область: Управление Взаимоотношениями с Клиентами (CRM - Customer Relationship Management) (Слайд 39):**
    *   CRM-системы рассматривают клиентов с многогранной точки зрения.
    *   Используют набор интегрированных приложений для охвата всех аспектов взаимоотношений с клиентами: обслуживание, продажи, маркетинг.
    *   Будет рассмотрена как часть систем планирования ресурсов предприятия (ERP).

#### 4. Информационные Системы Производства и Операций (Production and Operations Information Systems) (Слайды 40-56)

*   **Обзор (Слайд 40, 41):** Поддерживают управление, контроль и аудит производственных и операционных ресурсов.
*   **Компьютеры в Производстве (Слайд 42):**
    *   В производстве есть запасы: сырья, незавершенного производства, готовой продукции.
    *   Системы отслеживают **количество и затраты** по каждому виду запасов.
*   **Планирование Производственных Ресурсов (MRP - Manufacturing Resource Planning) (Слайды 43, 44):**
    *   **MRP (первоначальное значение Material Requirements Planning – Планирование Потребностей в Материалах):** Система, которая на основе спроса (главный производственный план, прогнозы, заказы) и предложения (запасы, заказы на закупку, производственные заказы) генерирует **Плановые Заказы (Planned Orders)** (что и когда нужно произвести или закупить) и **Сообщения о Действиях (Action Messages)** (например, ускорить/отложить заказ).
    *   **Ключевые входные данные для MRP (из схемы на слайде 44 и текста):**
        *   **Главный производственный план (Master Production Schedule - MPS):** Сколько готовой продукции нужно произвести и когда. Основан на заказах клиентов и прогнозах. (Стратегический уровень)
        *   **Спецификация материалов (Bill of Materials - BoM):** Список всех сырьевых материалов, компонентов и их количеств, необходимых для производства одной единицы готовой продукции.
        *   **Технологическая карта (Route Sheet):** Описывает последовательность операций, необходимых для производства детали/изделия, и определяет рабочие центры, где эти операции выполняются. Включает требуемые операции и стандартное время (люди + машины).
        *   **Данные о запасах:** Текущие остатки сырья, незавершенного производства, готовой продукции.
    *   **Эволюция от MRP к MRP II (Manufacturing Resource Planning) и затем к ERP (Enterprise Resource Planning) (Слайд 44, 56):** MRP II расширяет MRP, включая планирование мощностей, финансов и других ресурсов. ERP интегрирует все функции предприятия.
*   **Стратегические/Тактические/Операционные Системы в Производстве (Слайд 45):**
    *   **Главный Производственный План (Master Production Scheduling System):** Стратегический.
    *   **Система Планирования Потребностей в Материалах (MRP system):** Тактический. Включает BoM.
    *   **Система Планирования Потребностей в Мощностях (Capacity Requirements Planning - CRP system):** Тактический. Включает Технологическую карту (Route Sheet).
    *   **Детальный Производственный График (Detailed Production Schedule):** Операционный.
    *   **Управление Цехом (Shop Floor Control):** Операционный.
    *   **Контроль Качества (Quality Control):** Операционный.
    *   **Управление Запасами (Inventory Control):** Тактический и Операционный.
    *   **Учет Затрат (Cost Accounting):** Тактический и Операционный.
*   **Стратегические Производственные Системы (Слайд 46):**
    *   Помогают высшему руководству: выбирать место для завода, строить новый завод, проектировать производственные мощности, оценивать технологии для производственных процессов.
    *   Используют внутренние и внешние данные.
*   **MRP (Стратегическое) Планирование – Главный Производственный План (MPS) (Слайд 47):**
    *   Основан на: принятых заказах на продажу, прогнозе продаж, текущих запасах готовой продукции.
    *   Содержит перечень количества единиц продукции, которые нужно произвести каждую неделю.
*   **MRP (Тактическое) – Планирование Потребностей в Материалах (Слайд 48, 49):**
    *   Использует MPS и BoM для создания **поэтапных во времени заказов на закупку сырья** (главный выход MRP).
    *   **Bill of Materials (BoM) (Слайд 49):** Список сырья для производства одной единицы готовой продукции с указанием количества каждого материала. (Пример для обуви).
*   **MRP (Тактическое) – Планирование Потребностей в Мощностях (CRP) (Слайд 50):**
    *   **Технологическая карта (Route Sheet):** Показывает последовательность операций и стандартное время для каждой (человек + машина).
    *   Оценивает, достаточно ли машинного времени и рабочего времени. Может потребоваться аренда помещений/оборудования или найм временных рабочих.
    *   CRP генерирует **детальный производственный график** и выдает **производственные заказы** в цех.
*   **MRP (Операционное) – Планирование и Контроль Производства (Слайд 51):**
    *   Закупка сырья (когда, сколько).
    *   Потребности в оборудовании и рабочих.
    *   Детальные производственные графики.
    *   Сбор статистики для оценки (датчики, сканеры, терминалы в цеху).
    *   Контроль качества.
    *   Сравнение фактических данных с планами.
    *   Учет затрат на произведенную продукцию.
*   **ИТ-аспекты в Производстве (Слайд 52):**
    *   Большие БД для быстрого поиска.
    *   Сбор данных различными способами (датчики, измерительные приборы, сканирование).
    *   Связь (Connectivity) по всем производственным объектам.
    *   Системы как операционного, так и тактического уровня (например, управление цехом).
    *   Интеграция с системами вне производства.
*   **Программное обеспечение для Управления Цепочками Поставок (SCM - Supply Chain Management Software) (Слайд 53):**
    *   **Цепочка поставок (Supply chain):** Поток материалов, услуг и информации от поставщиков товаров и сырья через организацию к ее клиентам. Сейчас чаще говорят о **сети поставок (supply network)**.
    *   **Управление цепочками поставок (Supply chain management):** Процессы и процедуры для обеспечения доставки товаров и услуг клиентам по наименьшей стоимости при обеспечении наивысшей ценности для клиентов.
*   **Запасы, Управляемые Поставщиком (Vendor Managed Inventory - VMI) (Слайд 54):**
    *   Поставщики получают доступ к графикам производственного планирования организации, чтобы обеспечить возможность выполнения заказов.
    *   Производящая организация открывает свои системы клиенту, чтобы он мог видеть уровни запасов и производства перед размещением заказов.
*   **Производство "Точно в срок" (JIT - Just-In-Time Manufacturing) (Слайд 55):**
    *   Сырье поступает на производственную линию именно тогда, когда оно необходимо.
    *   Минимизирует запасы.
    *   Требует сложных операционных информационных систем.
    *   Может использовать запасы, управляемые поставщиком (компьютеры поставщика подключены к системам запасов компании).
*   **Эволюция MRP2 в ERP (Слайд 56):** Показывает, как системы управления финансами/бухгалтерией, управление взаимоотношениями с клиентами (CRM), управление цепочками поставок (SCM) и производственные модули (главный план, управление запасами, спецификации, рабочие заказы, закупки, маршрутизация) интегрируются в единую систему ERP.

**Ключевые моменты для запоминания из Лекции 5:**

*   **Финансовые ИС:** Поддержка тактических (бюджетирование, управление денежными средствами, капитальное бюджетирование, управление инвестициями) и стратегических (анализ фин. состояния, долгосрочное прогнозирование, корпоративное планирование) задач. Важность анализа коэффициентов и внешних данных.
*   **ИС Управления Персоналом (HRMS):** Охватывают операционные (зарплата, учет сотрудников, посещаемость, графики, оценка), тактические (контроль должностей, рекрутинг, компенсации) и стратегические (планирование рабочей силы, переговоры с профсоюзами) аспекты.
*   **Маркетинговые ИС:** Поддерживают операционные (управление контактами, SFA, телемаркетинг), тактические (формирование территорий продаж, распределение продавцов) и стратегические (прогнозирование продаж, маркетинговые исследования) задачи. Важность координации с другими системами и CRM.
*   **Производственные ИС (MRP/MRP II/ERP):** Ключевые компоненты – MPS, BoM, Route Sheet. Планирование потребностей в материалах (MRP) и мощностях (CRP). Концепции SCM, VMI, JIT. Эволюция к интегрированным системам ERP.

# Lesson 6: Enterprise Information Systems (SCM/CRM/ERP)

1.  **Цели обучения (Слайд 2):**
    *   Понять, как организации используют ИТ по всему предприятию.
    *   Объяснить поток товаров и услуг внутри организации.
    *   Описать системы SCM, CRM и ERP.
    *   Понять их связь с **Цепочкой создания ценности (Value Chain)**.

2.  **Категории Корпоративных Систем (Слайд 3):**
    *   **Системы масштаба предприятия (Enterprise-Wide Systems / Enterprise Systems):** Позволяют компаниям **интегрировать информацию** по всем операциям на уровне всей компании.
    *   **Межорганизационные системы (Inter-Organizational Systems - IOS):** Системы, которые "общаются" через границы организаций (например, с поставщиками, клиентами) с целью оптимизации потока информации между компаниями.

3.  **Планирование Ресурсов Предприятия (ERP) и Создание Ценности (Слайд 4):**
    *   **ERP (Enterprise Resource Planning) – центральная система** для организации.
    *   Обеспечивает **обмен информацией между всеми бизнес-функциями** и уровнями управления для поддержки работы и управления бизнесом.
    *   **Конечная цель:** Удовлетворение клиентов и получение **конкурентного преимущества** за счет снижения затрат и улучшения сервиса.
    *   **Создание ценности (Create Value):** Преобразование входов (ресурсов) в выходы (продукты/услуги), которые ценны для клиента. Организации создают ценность, выполняя действия дешевле или предлагая уникальные (дифференцированные) продукты/услуги. ИС помогают оптимизировать эти действия.

4.  **Цепочка Создания Ценности Бизнеса (Business Value Chain) (Слайды 5-8, 10-13):**
    *   **Определение (Слайд 5):** Управление материалами, услугами и информацией от поставщиков через производство, дистрибуцию до конечных потребителей.
    *   **Анализ цепочки создания ценности (Value Chain Analysis) (Портер, 1985, 2001) (Слайд 6):** Процесс анализа видов деятельности для определения, где добавляется ценность продуктам/услугам и какие затраты при этом возникают.
    *   **Компоненты цепочки создания ценности (Слайды 6, 7, 8, 10):**
        *   **Основные виды деятельности (Primary Activities) (Слайд 7):** Непосредственно создают и доставляют продукт/услугу.
            *   **Входящая логистика (Inbound Logistics):** Получение и складирование сырья, комплектующих.
            *   **Операции/Производство (Operations/Manufacturing):** Обработка заказов, превращение сырья в готовую продукцию.
            *   **Исходящая логистика (Outbound Logistics):** Дистрибуция готовой продукции клиентам.
            *   **Маркетинг и Продажи (Marketing and Sales):** Создание спроса на продукт (предпродажная деятельность).
            *   **Обслуживание клиентов (Customer Service):** Поддержка продукта или клиента (послепродажная деятельность).
        *   **Поддерживающие виды деятельности (Support Activities) (Слайд 8):** Обеспечивают выполнение основных видов деятельности.
            *   **Инфраструктура (Infrastructure):** Аппаратное и программное обеспечение для поддержки основных операций.
            *   **Управление персоналом (Human Resources):** Наем, планирование собеседований, управление льготами.
            *   **Развитие технологий (Technology Development):** Проектирование и разработка приложений, поддерживающих организацию.
            *   **Закупки (Procurement):** Приобретение товаров или услуг, необходимых для основных видов деятельности.
    *   **Роль ИС в цепочке создания ценности (Слайды 9, 10):** ИС играют значительную роль на всех этапах для достижения конкурентного преимущества.
        *   Должны **соответствовать бизнес-стратегии** (лидерство по затратам или дифференциация).
        *   Часто **интегрированы с реинжинирингом бизнес-процессов (BPR)** для улучшения операций.
    *   **Система ценности бизнеса – организационный фокус (Слайды 11-13):** Показывает, как цепочки ценности отдельных компаний (поставщик, фирма, дистрибьютор) связываются в общую систему ценности. Информация течет как "вверх по течению" (от клиента к поставщику), так и "вниз по течению" (от поставщика к клиенту).

5.  **Системы Управления Цепочками Поставок (SCM - Supply Chain Management Systems) (Слайды 14-22):**
    *   **Определение (Слайды 14, 16):** Управление материалами, услугами и информацией от поставщиков через производство, дистрибуцию до конечных клиентов.
    *   **Цепочка поставок (Supply Chain) (Слайд 16, 17, 18):** Поток материалов, услуг и информации от поставщиков товаров и сырья к клиентам. Включает прямых поставщиков и их поставщиков (сеть поставок).
    *   **Управление цепочками поставок (Supply Chain Management) (Слайд 16, 18):** Процессы и процедуры для обеспечения доставки товаров и услуг клиентам по наименьшей стоимости при обеспечении наивысшей ценности.
    *   **Цель SCM-систем (Слайд 17):** Приложения, ускоряющие разработку продукта и снижающие затраты, связанные с закупкой сырья, компонентов и услуг у поставщиков.
    *   **Источники SCM-систем (Слайд 17):** Могут быть от специализированных вендоров SCM-софта (Agile, Ariba, i2) или от вендоров ERP (SAP, Oracle). Тесно интегрируются с ERP.
    *   **Возможности SCM и ERP (на примере таблицы) (Слайд 19):**
        *   **Планирование (Planning):** Моделирование цепочек поставок, проектирование, планирование спроса и предложения, производственное и транспортное планирование.
        *   **Исполнение (Execution):** Интеграция планирования, обещания заказов, логистики и транзакционных систем (управление материалами, производством, отгрузкой, складом) с использованием технологии RFID.
        *   **Координация (Coordination):** Мониторинг и анализ процессов внутри и вне компании, управление событиями в цепи поставок и ее производительностью.
        *   **Сотрудничество (Collaboration):** Обмен информацией и достижение общих целей через совместное планирование, прогнозирование и пополнение запасов (CPFR), поддержка запасов, управляемых поставщиком (VMI), и запасов, управляемых поставщиком для поставщика (SMI).
    *   **Преимущества SCM (Слайд 20):**
        *   **Оптимизация рабочих процессов и повышение производительности сотрудников:** Эффективное управление командировками, временем, расходами; сотрудничество с поставщиками в реальном времени.
        *   **Ускорение разработки продукта:** Быстрая реакция на рыночные условия; сокращение времени вывода новых продуктов на рынок.
        *   **Снижение затрат и повышение эффективности в сети поставок:** Поддержка переговоров по контрактам; измерение и оптимизация эффективности соглашений с поставщиками.
    *   **Модели цепочки поставок: PUSH vs. PULL (Слайды 21, 22):**
        *   **PUSH-модель ("выталкивающая"):**
            *   Производство и дистрибуция планируются заранее, **на основе прогнозов спроса**.
            *   Поставщики производят товары в ожидании спроса.
            *   *Лозунг:* Поставка по прогнозу -> Производство по прогнозу -> Запасы по прогнозу -> Товар на полках по прогнозу -> Клиент покупает то, что есть на полках.
        *   **PULL-модель ("вытягивающая"):**
            *   Цепочка поставок управляется **фактическими заказами или покупками клиентов**.
            *   Производящая организация открывает свои системы клиенту, позволяя ему видеть запасы и уровни производства перед заказом.
            *   *Лозунг:* Поставка под заказ -> Производство под заказ -> Автоматическое пополнение склада -> Автоматическое пополнение запасов (у ритейлера) -> Клиент заказывает.

6.  **Системы Управления Взаимоотношениями с Клиентами (CRM - Customer Relationship Management Systems) (Слайды 23-40):**
    *   **Определение (Слайд 23):** Эффективное управление материалами, услугами и информацией от поставщиков к клиентам для улучшения отношений и стимулирования роста бизнеса. (Здесь определение немного смешано с SCM, но акцент на **улучшении отношений**).
    *   **Роль CRM в цепочке создания ценности (Слайд 24):** CRM-системы поддерживают в основном активности, связанные с маркетингом, продажами и обслуживанием, а также взаимодействуют с клиентами.
    *   **Функции CRM-систем (Слайд 25, 26):**
        *   **Собирают и интегрируют данные о клиентах** со всей организации.
        *   **Консолидируют и анализируют** эти данные.
        *   **Распространяют результаты** в различные системы и точки контакта с клиентами.
        *   **Предоставляют единую точку контакта** для клиента.
        *   Охватывают маркетинг (данные кампаний, контент, анализ данных), продажи (телефонные, веб, розничные, полевые) и сервис (данные колл-центра, веб-самообслуживание, беспроводной доступ).
    *   **Бизнес-ценность CRM (Слайд 27):**
        *   Повышение удовлетворенности клиентов.
        *   Более эффективный маркетинг и снижение затрат на прямой маркетинг.
        *   Снижение затрат на привлечение и удержание клиентов.
        *   Увеличение выручки за счет выявления наиболее прибыльных клиентов и сегментов для маркетинга, кросс-продаж (cross-selling) и допродаж (up-selling).
        *   Снижение **коэффициента оттока клиентов (churn rate)** (количество клиентов, прекративших пользоваться продуктами/услугами).
    *   **Операционный и Аналитический CRM (Слайд 28):**
        *   **Операционный CRM (Operational CRM) (Слайды 29, 30):** Приложения, ориентированные на клиента (customer-facing), такие как автоматизация отдела продаж (SFA), колл-центр, поддержка обслуживания клиентов, автоматизация маркетинга.
            *   *Примеры:* Управление кампаниями, программы лояльности (Air Miles), е-маркетинг, управление контактами, управление лидами, телемаркетинг, электронные продажи.
            *   *Пример процесса управления лояльностью (Слайд 30):* Запрос на обслуживание -> Получение информации о клиенте -> Оценка клиента (скоринг) -> Если ценный/лояльный -> Специальные предложения; иначе -> Маршрутизация к лучшему агенту -> Решение проблемы.
        *   **Аналитический CRM (Analytical CRM) (Слайды 31-34):** Приложения, которые анализируют данные о клиентах, собранные операционными CRM-приложениями, для получения информации, улучшающей эффективность бизнеса.
            *   *Примеры:* Разработка стратегий сегментации клиентов и профилей клиентов; анализ прибыльности клиентов или продуктов; выявление тенденций в длительности цикла продаж; анализ сгенерированных лидов и коэффициентов конверсии.
            *   *Архитектура (Слайд 32):* Данные из различных каналов (колл-центр, веб-сайт, продажи, email и т.д.) и других источников (унаследованные системы, демография) поступают в **хранилище данных о клиентах (Customer data warehouse)**. Затем с помощью инструментов OLAP, Data Mining и др. анализируются для выявления прибыльных клиентов, сегментов рынка, профилей клиентов, коэффициентов оттока.
            *   *Аналитика помогает (Слайд 33):* Выявлять самых важных клиентов, прогнозировать будущие модели покупок, правильно распределять ресурсы для увеличения продаж.
            *   **Измерение производительности CRM (Слайд 34):**
                *   Метрики: стоимость лида, стоимость продажи, количество повторных клиентов, снижение оттока, коэффициент закрытия продаж.
                *   **Пожизненная ценность клиента (Customer Lifetime Value - CLTV):** Разница между доходами и расходами от клиента, за вычетом затрат на маркетинговое продвижение для удержания клиента.
    *   **Программное обеспечение CRM (Слайд 35):** От нишевых инструментов до крупных корпоративных приложений. Может связываться с другими корпоративными приложениями, такими как SCM.
    *   **Интегрированный портал CRM (Слайд 37):** Центральное хранилище данных (InterAction Database), которое собирает информацию из различных источников (внешний контент, бухгалтерские/HR системы, списки рассылки, личные контакты) и предоставляет ее через различные клиенты (Windows, веб, мобильные устройства).
    *   **Расширение корпоративного ПО (Слайд 38):** CRM-системы становятся более веб-ориентированными, работают с расширенными цепочками поставок, новыми моделями B2C и B2B электронной коммерции.
    *   **Сервисные платформы и Управление бизнес-процессами (BPM) (Слайды 38, 39):** Интеграция нескольких приложений из разных функций для обеспечения бесшовного опыта для клиента, сотрудника, менеджера или партнера. BPM – непрерывное управление и оптимизация бизнес-процессов.
    *   **Порталы (Слайд 39):** Платформы для создания комплексных сервисов и интеграции информации из корпоративных приложений и унаследованных систем.
    *   **Возможности и рекомендации для CRM (Слайд 40):**
        *   **Возможности:** Улучшение координации процессов и принятия решений; снижение затрат на запасы, времени от заказа до доставки; более эффективное реагирование на запросы клиентов; повышение прибыльности продуктов и клиентов.
        *   **Рекомендации по внедрению:** Сначала смотреть на бизнес-цели; уделять внимание управлению данными; заручиться поддержкой высшего руководства и сотрудников; обеспечить обучение.

7.  **Планирование Ресурсов Предприятия (ERP - Enterprise Resource Planning) (Слайды 41-49):**
    *   **Определение (Слайд 43):** Метод эффективного планирования и контроля всех ресурсов, необходимых для принятия, изготовления, отгрузки и учета заказов клиентов в производственной, дистрибьюторской или сервисной компании.
    *   **Роль ERP в цепочке создания ценности (Слайд 42):** ERP-системы охватывают и интегрируют практически все основные и поддерживающие виды деятельности в цепочке создания ценности.
    *   **Интегрированные пакеты (Слайд 44):** Многофункциональные системы, предназначенные для поддержки многих организационных функций (например, бухгалтерия и финансы).
    *   **Ключевые характеристики ERP (Слайд 44):**
        *   **Внутренне ориентированные:** Поддерживают внутренние операции организации.
        *   **Высокоинтегрированные:** Используют общее хранилище данных для обмена информацией между функциями, с обновлениями в реальном времени.
        *   **Организационное соответствие:** Может быть не идеальным для отдельных отделов, но интегрированный обмен информацией обычно перевешивает этот недостаток.
        *   **Обычно пакетные приложения:** Поддерживаются вендором, используют общий пользовательский интерфейс.
        *   **Кастомизация не приветствуется:** Но системы предлагают гибкость для поддержки внешних приложений через общее хранилище данных и интерфейсы.
    *   **Архитектура ERP-системы (Слайд 45):** **Централизованная база данных** является ядром, к которому подключены модули для различных функций: Финансы и Бухгалтерия, Продажи и Маркетинг, Производство, Управление персоналом.
    *   **Программное обеспечение ERP (Слайд 46):**
        *   Взаимозависимые программные модули с общей центральной базой данных.
        *   Поддерживают основные внутренние бизнес-процессы.
        *   Позволяют использовать данные несколькими функциями и бизнес-процессами для точной координации и контроля.
        *   Разработаны на основе **предопределенных бизнес-процессов** (часто "лучших практик" - best practices).
        *   Компании выбирают необходимые функции, а затем сопоставляют их с предопределенными процессами в ПО.
        *   **Лучшие практики (Best practices):** Наиболее успешные решения или методы решения проблем для последовательного достижения цели.
    *   **ERP-система SAP (Слайды 47-49):**
        *   Один из ведущих мировых поставщиков ERP (родом из Германии).
        *   Поддержка международных транзакций и мультинациональных компаний.
        *   Работает на различных СУБД и аппаратных платформах.
        *   Подходит для крупных и малых компаний.
        *   Дорогая, но цена относительна (учитывая потенциальную выгоду).
        *   **Модули SAP (Слайд 47):** Финансы, Логистика, Управление персоналом.
        *   **Возможности SAP (примеры) (Слайды 48, 49):**
            *   **Бизнес-анализ:** Оценка производительности бизнеса (персонал, операции, цепочка поставок).
            *   **Финансовый и управленческий учет:** Автоматизация управления финансовыми потоками, фин. учета, управленческого учета.
            *   **Управление человеческим капиталом:** Инструменты для максимизации потенциала персонала (управление транзакциями сотрудников, жизненный цикл сотрудника).
            *   **Управление корпоративными услугами:** Оптимизация управления недвижимостью, командировками, поощрениями, комиссионными.
            *   **Самообслуживание:** Портал для сотрудников и менеджеров для создания, просмотра и изменения ключевой информации через веб, голосовые и мобильные устройства.

**Ключевые моменты для запоминания из Лекции 6:**

*   **Корпоративные системы (Enterprise Systems)** интегрируют информацию по всей компании. **Межорганизационные системы (IOS)** связывают компании друг с другом.
*   **Цепочка создания ценности (Value Chain):** Основные и поддерживающие виды деятельности, которые добавляют ценность продукту/услуге. ИС играют роль на каждом этапе.
*   **SCM (Управление цепочками поставок):** Оптимизация потока материалов, информации и услуг от поставщиков к клиентам. Модели PUSH (по прогнозу) и PULL (по заказу).
*   **CRM (Управление взаимоотношениями с клиентами):** Сбор, анализ и использование данных о клиентах для улучшения маркетинга, продаж и обслуживания. Операционный CRM (взаимодействие с клиентом) и Аналитический CRM (анализ данных о клиентах, CLTV).
*   **ERP (Планирование ресурсов предприятия):** Интегрированная система, охватывающая все основные бизнес-процессы и функции организации на основе общей центральной базы данных и предопределенных "лучших практик".
*   **Взаимосвязь SCM, CRM и ERP:** Это крупные корпоративные системы, которые часто интегрируются друг с другом для обеспечения сквозных бизнес-процессов и потоков информации. ERP часто является ядром, а SCM и CRM – расширениями, фокусирующимися на внешних взаимодействиях (с поставщиками и клиентами соответственно).

# Lesson 7: Systems that Span Organizational Boundaries

1.  **Цели обучения (Слайд 2):**
    *   Описать характеристики шести информационных систем, которые охватывают организационный, управленческий и исполнительный уровни:
        *   **Системы автоматизации офиса (OAS - Office Automation Systems)**
        *   **Технологии для совместной работы (Collaboration Technologies)**
        *   **Глобальные (Географические) информационные системы (Global/Geographic Information Systems - GIS)**
        *(Примечание: в целях указано шесть систем, но далее подробно рассматриваются в основном эти три категории. Возможно, остальные (DSS, Expert Systems, Functional Area IS) упоминаются как часть общей картины на схеме).*

2.  **Системы, выходящие за рамки организационных границ (Слайд 4):**
    *   На схеме показано, как различные типы ИС (Системы поддержки принятия решений - DSS, Экспертные системы - ES, Системы автоматизации офиса - OAS, Технологии для совместной работы, Функциональные ИС, Географические ИС - GIS) обслуживают все три уровня управления (операционный, управленческий, исполнительный) и помогают им взаимодействовать.
    *   Эти системы ориентированы на решения разной степени структурированности: от структурированных на операционном уровне до неструктурированных на исполнительном.

#### I. Системы Автоматизации Офиса (OAS - Office Automation Systems) (Слайды 3, 5-9)

*   **Что это? (Слайд 5):** Компьютеризация и интеграция офисных задач с помощью технологий. Вместо "перемалывания цифр" (number crunching) они часто занимаются "перемалыванием документов" (document crunching).
*   **Определение (Слайд 6):** Набор программного и аппаратного обеспечения, используемый для повышения производительности в офисе через сбор, хранение и обработку офисной информации, необходимой для выполнения основных задач и достижения целей.
*   **Примеры деятельности, поддерживаемой OAS (Слайд 6):**
    *   Создание документов или бизнес-форм на основе данных из других приложений или баз данных.
    *   Создание презентаций на основе внешних данных.
    *   Автоматическая отправка электронных писем клиентам или группам.
    *   Создание пользовательских механизмов ввода данных.
    *   Поддержка и организация данных, хранящихся в электронных таблицах или базах данных.
    *   Создание отдельных программ для автоматизации офисной среды.
*   **Другие поддерживаемые виды деятельности (Слайд 7):**
    *   **Планирование ресурсов:** Электронные календари с управлением ресурсами (оборудование, помещения и т.д.).
    *   **Коммуникации:** Электронная почта, голосовая почта, видеоконференции, **groupware** (ПО для совместной работы).
    *   **Системы обработки изображений (Imaging systems):** Преобразование фотографий и диаграмм в серию точек и их передача в магнитном виде на диск для хранения.
*   **Причины роста популярности OAS (Слайд 8):**
    *   Ценность информации и информационный взрыв (данных становится все больше).
    *   Рост офисных затрат и необходимость повышения производительности офиса.
    *   Доступность оборудования и навыков.
    *   Преимущества для управления человеческими ресурсами.
*   **Архитектура OAS (Слайд 9):**
    *   **Вход (Input):** Документы, графики, данные.
    *   **Процесс (Process):** Программы OAS (хранение, слияние, вычисление, передача). Используют базы данных OAS.
    *   **Выход (Output):** Сообщения, отчеты, графики.

#### II. Географические Информационные Системы (GIS - Geographic Information Systems) (Слайды 10-18)

*   **Что это? (Слайд 12):** Система, которая **связывает данные с физическими местоположениями**.
    *   Представляет данные на карте в различных форматах.
    *   Может отражать демографическую информацию в дополнение к географической.
    *   Может использовать информацию со спутников GPS.
*   **На какие вопросы "ГДЕ?" помогают отвечать GIS (Слайд 14):**
    *   Кратчайшее расстояние для грузовика для доставки товаров.
    *   Эффективные маршруты для школьных автобусов.
    *   Где разместить полицейские участки.
    *   Где бурить на нефть.
    *   Формирование территорий продаж.
*   **Из чего состоит типичная GIS (Слайд 15, 16):**
    *   База данных количественных и качественных данных.
    *   База данных карт.
    *   Программа, которая отображает информацию на картах.
    *   Оцифрованные карты создаются на основе спутниковых и аэрофотоснимков, а также табличной информации.
*   **GIS для принятия решений (Слайды 17, 18):**
    *   **Пример с супермаркетом:** Система показывает распределение населения по возрасту и доходу на карте города. Руководство решает, где строить следующий магазин.
    *   **Пример с полицией:** Карты с указанием мест совершения конкретных преступлений. Помогают решить, сколько полицейских машин направить в разные районы.
    *   **Пример с правительством и осадками:** Определение мест выпадения осадков. Сравнение этой информации с расположением болот может показать, какие болота получают мало осадков и могут высохнуть. Это помогает принимать решения о законодательстве по взаимодействию с болотами.

#### III. Технологии для Совместной Работы (Collaboration Technologies / Groupware) (Слайды 19-28)

*   **Коллаборативные Информационные Системы (Слайд 21):**
    *   Системы, которые позволяют группам людей достигать общей цели путем улучшения и облегчения коммуникаций и обмена знаниями.
    *   Используют сетевые технологии: телеконференции, совместное использование документов, данных, информации.
    *   *Пример:* Каждый участник может анонимно подавать идеи, которые отображаются на большом экране для обсуждения.
    *   Могут включать системы поддержки групповых решений (GDSS).
    *   Считаются **"зеленой технологией" (Green Technology)** – почему? (Вероятно, потому что уменьшают необходимость в поездках, экономя топливо и снижая выбросы).
*   **Технологии для совместной работы (Groupware) / Системы Поддержки Групп (GSS - Group Support Systems) (Слайд 22):**
    *   Программное обеспечение, позволяющее людям работать вместе более эффективно.
    *   **Два типа:**
        *   **Асинхронное Groupware (Asynchronous Groupware):** Системы, которые **не требуют** одновременной работы пользователей.
            *   *Примеры:* Электронная почта, новостные группы, автоматизация рабочих процессов (workflow automation), групповые календари, инструменты для совместного написания текстов.
        *   **Синхронное Groupware (Synchronous Groupware):** Системы, которые позволяют и поддерживают **одновременное** взаимодействие в группе.
            *   *Примеры:* Общие электронные доски (shared whiteboards), системы поддержки электронных совещаний, системы видеосвязи.
*   **Технологии видеоконференцсвязи (Слайд 23):**
    *   ПО и оборудование, позволяющие сторонам встречаться электронно (изображение и голос).
    *   **Автономная видеоконференцсвязь (Stand-alone Videoconferencing):** Высококачественные, обычно дорогие системы со специальными микрофонами, камерами и оборудованием. Могут поддерживать встречи нескольких людей из разных мест одновременно.
    *   **Настольная видеоконференцсвязь (Desktop Videoconferencing):** Более низкое качество, относительно недорогие системы, использующие ПК, небольшую камеру и микрофон (или телефон для голосовой связи). Позволяют общаться двум людям с рабочих мест.
    *   **Технология Telepresence:** Создает эффект присутствия (например, Cisco Telepresence Magic).
*   **Системы Поддержки Групп (GSS) (Слайд 24, 25):**
    *   ПО и оборудование для эффективной поддержки принятия групповых решений.
    *   Также называются **системами поддержки принятия групповых решений (group decision support system)** или компьютеризированными системами совместной работы.
    *   **Инструменты GSS (Слайд 25):** E-mail, мгновенные сообщения (IM), текстовые сообщения (TM), видеоконференции, групповое планирование, управление проектами, совместное использование документов.
*   **Альтернативы GSS (Слайд 26):**
    *   **Комната для принятия решений (Decision room):** Специально оборудованное помещение для поддержки принятия решений, где участники находятся в одном здании.
    *   **Локальная сеть для принятия решений (Local area decision network):** Участники группы находятся в одном здании или географической области; групповое принятие решений происходит часто.
*   **Пример коллаборативной системы (Слайд 27):** ThinkTank TM (www.groupsystems.com) – инструмент для бизнес-сотрудничества (поддержка групповых решений). Помогает в мозговом штурме, организации, приоритизации, оценке, выявлении и документировании инновационного процесса. Группы могут находиться в одной комнате или быть распределены на большие расстояния.
*   **Примеры сервисных приложений с использованием коллаборативных технологий (Слайд 28):**
    *   Профессиональные услуги – Офис будущего (Microsoft).
    *   Розничная торговля – Будущее шопинга (Cisco).
    *   Банковское дело – Будущее банковского дела (Microsoft).
    *   Здравоохранение – Будущее здравоохранения (Microsoft).
    *   OPENPediatrics (IBM) – Больницы без стен.

**Ключевые моменты для запоминания из Лекции 7:**

*   **Системы, выходящие за рамки организационных границ,** облегчают работу и принятие решений на всех уровнях организации и между организациями.
*   **Системы автоматизации офиса (OAS):** Повышают производительность офисных задач, связанных с документами, коммуникациями, планированием.
*   **Географические информационные системы (GIS):** Связывают данные с картами, помогая отвечать на вопросы "где?" и принимать решения, основанные на местоположении (выбор места для магазина, планирование маршрутов, анализ криминогенной обстановки).
*   **Технологии для совместной работы (Collaboration Technologies / Groupware / GSS):** Позволяют группам эффективно работать вместе, обмениваться информацией и принимать решения, независимо от того, находятся ли участники в одном месте или удаленно.
    *   **Асинхронные** (e-mail, календари) – не требуют одновременной работы.
    *   **Синхронные** (видеоконференции, электронные доски) – для одновременного взаимодействия.
    *   **Видеоконференции и Telepresence** – для виртуальных встреч.
    *   **GSS** – для поддержки группового принятия решений.

# Lesson 8: Systems that Span Organizational Boundaries: Decision Support, Expert and Knowledge Management systems

#### I. Системы Поддержки Принятия Решений (DSS - Decision Support Systems) (Слайды 3-31)

*   **Что это? (Слайд 3):** Компьютеризированные информационные системы, разработанные для помощи владельцам бизнеса, руководителям и менеджерам в решении **сложных бизнес-проблем** и/или ответе на вопросы.
*   **Типы решений (Слайд 3):**
    *   **Структурированные решения (Structured Decisions):** Принимаются на основе четко заданных параметров. Известно, какие данные нужны, и предполагается, что они всегда доступны. Мало гибкости для лица, принимающего решение (ЛПР). (Чаще на операционном уровне).
    *   **Неструктурированные решения (Unstructured Decisions):** Принимаются, когда не все элементы бизнес-среды полностью понятны (например, вывод нового продукта, маркетинговая стратегия). Данные могут быть неполными. Больше фокуса на индивидууме, принимающем решение. (Чаще на стратегическом уровне).
    *   *(Существуют также **полуструктурированные решения** – комбинация обоих типов, где часть проблемы структурирована, а часть нет. DSS часто помогают именно с такими).*
*   **DSS – определение и детали (Слайд 4):**
    *   **Информационные системы специального назначения**, разработанные для поддержки сотрудников управленческого уровня в принятии организационных решений.
    *   Используют **вычислительное ПО (часто MS Excel)** для построения **моделей** для анализа и решения проблем (например, прогнозирование продаж или ресурсов).
    *   **Поддерживаемая деятельность: Анализ "что-если" ("What-if" analysis)** – изменение одной или нескольких переменных в модели для наблюдения за эффектом (например, "Каким будет платеж, если процентная ставка вырастет на 1%?").
*   **Характеристики DSS (Слайд 5):**
    *   **Вход (Inputs):** Данные и модели; команды ввода и обработки данных (через пользовательский интерфейс).
    *   **Обработка (Processing):** Интерактивная обработка данных и моделей; моделирование (simulations), оптимизация, прогнозы.
    *   **Выход (Outputs):** Графики и текстовые отчеты; обратная связь оператору системы (через пользовательский интерфейс).
    *   **Типичные пользователи (Typical Users):** Менеджеры среднего звена (хотя DSS может использоваться на любом уровне организации).
*   **Примеры моделей DSS по областям (Слайд 6):** (Очень подробная таблица, основные идеи)
    *   **Бухгалтерия:** Анализ затрат, анализ безубыточности, бюджетирование.
    *   **Корпоративный уровень:** Корпоративное планирование, анализ слияний и поглощений.
    *   **Финансы:** Анализ денежных потоков, рентабельность инвестиций, бюджетирование капиталовложений, управление портфелем.
    *   **Маркетинг:** Прогноз спроса на продукт, анализ рекламной стратегии, ценообразование.
    *   **Персонал:** Переговоры по трудовым вопросам, анализ рынка труда, оценка навыков персонала.
    *   **Производство:** Проектирование продукта, производственное планирование, анализ расположения завода.
    *   **Наука управления (Management Science):** Линейное программирование, деревья решений, имитационное моделирование.
    *   **Статистика:** Регрессионный и корреляционный анализ, проверка гипотез.
*   **Дополнительные характеристики DSS (Слайд 7):**
    *   Могут разрабатываться людьми **вне IT-отдела** (например, самими менеджерами с помощью Excel).
    *   Могут иметь возможность **ad-hoc отчетности** из базы данных (хранилища).
    *   **Примеры поддержки решений:** "Стоит ли нам покупать компанию?", "Стоит ли расширяться на новую продуктовую линейку?" (Это примеры полуструктурированных решений).
*   **Компоненты DSS (Слайды 8-17):**
    1.  **Система управления базами данных (СУБД / DBMS) (Слайды 10, 11):**
        *   Позволяет менеджерам и ЛПР выполнять качественный анализ данных, хранящихся в базах данных компании, хранилищах данных и витринах данных.
        *   Может использоваться для подключения к внешним базам данных.
        *   **DSS, управляемая данными (Data-driven DSS):** Выполняет качественный анализ на основе баз данных компании.
    2.  **База моделей (Model base) (Слайды 12, 13):**
        *   Позволяет менеджерам и ЛПР выполнять **количественный анализ** внутренних и внешних данных.
        *   **ПО для управления моделями (Model management software - MMS):** Координирует использование моделей в DSS.
        *   **DSS, управляемая моделями (Model-driven DSS):** Выполняет математический или количественный анализ.
        *   **Примеры типов моделей (Слайд 14):**
            *   **Финансовые:** Анализ денежных потоков, внутренняя норма рентабельности (IRR). ПО: Excel.
            *   **Статистические:** Сводная статистика, прогнозы тенденций, проверка гипотез. ПО: SPSS, SAS.
            *   **Графические:** Помогают ЛПР в проектировании, разработке и использовании графического отображения данных. ПО: PowerPoint.
            *   **Управление проектами:** Обрабатывают и координируют крупные проекты. ПО: MS Project.
    3.  **Пользовательский интерфейс или Менеджер диалога (User Interface or Dialogue Manager) (Слайд 15, 16):**
        *   Позволяет ЛПР легко получать доступ к DSS и управлять ею, используя обычные бизнес-термины.
        *   Позволяет пользователям взаимодействовать с DSS для получения информации.
        *   Обеспечивает все аспекты коммуникации между пользователем и аппаратно-программным обеспечением DSS.
        *   Позволяет манипулировать переменными.
    *   **Общая схема трех компонентов DSS (Слайд 17):** Пользователь взаимодействует через **Интерфейс (User Interface Management)**, который обращается к **Управлению Моделями (Model Management)** (выбор модели, результаты моделирования) и **Управлению Данными (Data Management)** (необходимая информация). Модели могут быть "что-если", оптимизационные, целеполагающие, статистические. Данные могут быть организационными, внешними, персональными.
*   **DSS, управляемая моделями (Model-Driven DSS) vs. DSS, управляемая данными (Data-Driven DSS) (Слайды 18-23):**
    *   **Model-Driven DSS (Слайды 18, 19, 20, 21):** Использует различные модели (статистические, имитационные, финансовые) для принятия решений. Решения основаны на моделях.
        *   Примитивный пример: электронная таблица для анализа "что-если".
        *   Шаблоны Excel для определенных типов решений.
        *   **Шаблон (Template):** Рамка для общего определения аспекта реальности. Экземпляры создаются заполнением шаблона.
        *   **Модель (Model):** Представление реального процесса (графическое или числовое). Содержит переменные, которые можно изменять.
        *   *Пример:* Кредитный калькулятор в Excel (Слайд 21).
    *   **Data-Driven DSS (Слайды 18, 22):** Акцентирует внимание на доступе и обработке временных рядов внутренних данных компании (иногда внешних) для помощи в принятии решений. Решения основаны на проанализированных данных.
        *   Данные из различных источников -> Хранилище данных/Куб -> Аналитика/Дашборд -> Пользователи (руководство, менеджеры).
    *   **Сравнительная таблица Model-Driven vs. Data-Driven DSS (Слайд 23):**
        *   **Взаимодействие:** Model-Driven – с моделью; Data-Driven – с данными.
        *   **Проблемы:** Model-Driven – хорошо определенные, структурированные (что-если); Data-Driven – в основном неструктурированные.
        *   **Модели:** Model-Driven – сложные; Data-Driven – простые.
        *   **Данные:** Model-Driven – большие объемы не обязательны; Data-Driven – большие объемы критичны.
        *   **Помощь:** Model-Driven – понять влияние решений; Data-Driven – подготовить решения, показывая прошлое и выявляя закономерности.
        *   **Технология:** Model-Driven – настольное ПО (Excel, Access); Data-Driven – запросные приложения к корпоративной БД/хранилищу.
*   **Сравнение DSS и MIS (Слайды 24, 25):** (Детальная таблица на слайде 25)
    *   **Тип проблемы:** DSS – неструктурированные; MIS – только структурированные.
    *   **Пользователи:** DSS – индивидуумы, малые группы, вся организация; MIS – в основном организация.
    *   **Поддержка:** DSS – все аспекты и фазы принятия решений, не заменяет ЛПР; MIS – в некоторых случаях автоматизирует решения.
    *   **Акцент:** DSS – на решениях и стилях принятия решений; MIS – на информации.
    *   **Подход:** DSS – прямая поддержка с интерактивными отчетами; MIS – косвенная поддержка через регулярные отчеты.
    *   **Система:** DSS – онлайн, реальное время; MIS – печатные отчеты, доставляемые периодически.
    *   **Скорость:** DSS – гибкая, быстрая разработка; MIS – время отклика обычно дольше.
    *   **Выход:** DSS – экранные отчеты; MIS – печатные отчеты и документы.
    *   **Разработка:** DSS – пользователи более вовлечены; MIS – часто разрабатывается давно, пользователи не так вовлечены.
*   **Веб-ориентированные примеры DSS для конечных клиентов (Слайд 26):**
    *   **Model-Driven:** Fidelity.com (оценка инвестиций в ипотечные портфели).
    *   **Data-Driven:** Travelocity.ca, Expedia.ca (оценка и сравнение авиабилетов); Edmunds.com (оценка и сравнение цен на автомобили).
*   **Другие примеры Data-Driven DSS (Слайды 27-30):**
    *   **Планировщик защиты урожая в прериях (Канада):** Фермер вводит данные (оборудование, размер поля, цены на химикаты), модель рассчитывает нормы внесения, затраты, количество химикатов. (Слайд 27)
    *   **Помощь фермерам в Небраске (США):** Решение, где сажать виноград, чтобы избежать заморозков. (Слайд 27)
    *   **Авиационная отрасль (Слайды 28, 29, 30):** DSS помогает определить цены для максимизации выручки с каждого рейса (системы управления доходностью - yield management). Менеджер вводит данные (аэропорты, остановки, время, кол-во дней до вылета, кол-во пассажиров, размер самолета, загрузка прошлых рейсов), система предлагает переменные цены на билеты. Это объясняет, почему цена на билет может измениться за час.

#### II. Экспертные Системы (ES - Expert Systems) (Слайды 32-46)

*   **Что это? (Слайд 33):** Системы специального назначения, используемые сотрудниками операционного уровня для принятия решений, которые обычно принимаются более опытными сотрудниками или экспертами в данной области.
*   **Детали системы (Слайд 33):** Используют **механизмы логического вывода (inference engines)**, которые сопоставляют факты и правила, задают вопросы пользователю, делают заключение и представляют рекомендацию.
*   **Поддерживаемые виды деятельности (Слайд 33):** Медицинская диагностика, конфигурация оборудования, финансовое планирование, помощь в работе с программными приложениями (мастера помощи - help wizards).
*   **Архитектура ES (Слайд 34):**
    *   **Вход (Input):** Запрос о помощи, ответы на вопросы.
    *   **Процесс (Process):** Программы ES (сопоставление с образцом - pattern matching). Используют **Базу Знаний (Knowledge Base)** фактов и правил.
    *   **Выход (Output):** Рекомендации и советы.
*   **Пример диалога с ES (Слайды 35, 36, 37):** Система ведет пользователя через серию вопросов для рекомендации ресторана.
    *   Особенность ES: диалоги **не запланированы заранее**, синтезируются на основе текущей информации и базы знаний. Пользователь может не отвечать на вопрос, и это не остановит консультацию.
    *   **Система объяснений (Explanation system):** ES может объяснить свою логику (например, на вопрос "Почему?" система объясняет, как она пришла к выводу о типе ресторана). (Слайд 37)
*   **Отличия ES от традиционных систем (Слайд 38, 39):**
    *   Применяют **искусственный интеллект (AI)** к ситуациям со множеством фактов и сложными правилами принятия решений.
    *   Имитируют мышление эксперта.
    *   Манипулируют **знаниями (knowledge)**, а не просто информацией.
    *   Используют множество правил "ЕСЛИ... ТО..." (IF...THEN rules) – способ кодирования знаний.
    *   Должны уметь **объяснять свою логику** пользователю.
*   **Зачем разрабатывать ES? (Слайд 40):**
    *   Сохранить знания эксперта, если он уходит на пенсию или умирает.
    *   Объединить экспертизу нескольких экспертов.
    *   "Клонировать" знания эксперта и сделать их доступными во многих местах одновременно (например, лечение рака в удаленных районах).
    *   Могут разрабатываться с помощью детального программирования или через "оболочки экспертных систем" (expert system shell), например, VP Expert.
*   **Структура ES (Слайд 41, 42):**
    *   **База знаний (Knowledge base):** Факты и правила.
    *   **Механизм вывода (Inference engine):** ПО, которое принимает ввод пользователя и "просеивает" базу знаний, имитируя мышление эксперта. Это и есть **Искусственный Интеллект (AI)**.
    *   **Компоненты (Слайд 42):** База знаний, Механизм вывода, Средство пополнения базы знаний (от экспертов), Пользовательский интерфейс, Средство объяснения.
*   **Разработка ES (Слайды 43, 44):**
    *   **Инженер по знаниям (Knowledge Engineer):** Специалист по извлечению информации и экспертизы от экспертов.
    *   Переводит знания эксперта в набор правил "ЕСЛИ... ТО...".
    *   *Пример правил для системы выдачи кредитов (Слайд 44).*
*   **Примеры ES (Слайд 45):** Parker (приложение для парковки), AMDOCS (система для колл-центра).
*   **IBM Watson для здравоохранения (Слайд 46):** Анализирует данные пациентов, результаты тестов, медицинские исследования для формирования и проверки гипотез о диагнозах, предлагая список потенциальных диагнозов с уровнем уверенности.

#### III. Системы Управления Знаниями (KMS - Knowledge Management Systems) (Слайды 47-55)

*   **Определения (Слайд 47, 50):**
    *   **Управление знаниями (Knowledge Management - KM):** Процесс, используемый организацией для получения максимальной ценности от своих **активов знаний**.
    *   **Активы знаний (Knowledge Assets):** Все базовые навыки, рутины, практики, принципы, формулы, методы, эвристики и интуиция, как явные, так и неявные.
    *   **Явные знания (Explicit Knowledge):** Все, что может быть документировано, заархивировано, измерено или кодифицировано, часто с помощью ИС.
    *   **Неявные (Скрытые) знания (Tacit Knowledge):** Процессы и процедуры эффективного выполнения конкретной задачи, хранящиеся в уме человека.
    *   **Данные (Data):** Сырые факты.
    *   **Информация (Information):** Организованный набор фактов, имеющий дополнительную ценность.
    *   **Знание (Knowledge):** Осознание и понимание набора информации и способов ее полезного использования для решения задачи или принятия решения.
    *   **Система управления знаниями (KMS):** Организованный набор людей, процедур, ПО, баз данных и устройств, используемый для создания, хранения, совместного использования и применения знаний и опыта организации.
*   **Связь ES и KMS (Слайд 48):** Экспертная система работает на базе знаний и является частью более широкой области управления знаниями. *Процесс:* База данных (сырые факты) -> Информация и поддержка решений (информация) -> База знаний (закономерности и связи). (Повышение уровня понимания).
*   **KMS (Слайд 49):**
    *   **Лучшие практики (Best Practices):** Процедуры и процессы, широко признанные как наиболее эффективные и/или действенные.
    *   **Основная цель KMS:** Распознавать, генерировать, хранить, делиться и управлять этими неявными знаниями (лучшими практиками) для их применения.
    *   **Технология:** Обычно не одна технология, а набор инструментов: коммуникационные (e-mail, groupware, IM), системы хранения и поиска информации (СУБД).
*   **Разница между Данными, Информацией и Знаниями (Слайд 51):**
    *   **Данные:** "На складе розничного магазина 20 ПК".
    *   **Информация:** "Запасы в магазине закончатся через неделю, если сегодня не заказать еще".
    *   **Знание:** "Позвоните по номеру 800-555-2222, чтобы заказать еще запасы".
*   **Работники с данными и знаниями, Сообщества практиков (Слайды 52, 53):**
    *   **Работники с данными (Data workers):** Секретари, административные ассистенты, бухгалтеры.
    *   **Работники со знаниями (Knowledge workers):** Создают, используют и распространяют знания. Профессионалы в науке, инженерии, бизнесе.
    *   **Главный директор по знаниям (CKO - Chief Knowledge Officer):** Руководитель высшего звена, помогающий организации использовать KMS для достижения целей.
    *   **Сообщества практиков (COP - Communities of Practice):** Группы людей, объединенные общей дисциплиной или практикой. Могут использоваться для создания, хранения и обмена знаниями.
*   **Получение, хранение, совместное использование и применение знаний (Слайды 54, 55):**
    *   Работники со знаниями часто работают в командах.
    *   **Хранилище знаний (Knowledge repository):** Включает документы, отчеты, файлы, базы данных.
    *   **Карта знаний (Knowledge map):** Каталог, который направляет работника со знаниями к нужным знаниям.
    *   **Корпоративная KMS (Слайд 55):** Имеет возможности для классификации, организации и управления структурированными и полуструктурированными знаниями, делая их доступными по всему предприятию. Пользователи -> (Создание/Захват, Тегирование, Хранение/Извлечение, Управление/Обзор, Распространение/Публикация) -> Единое хранилище (отчеты, лучшие практики, заметки, презентации, email, графика, видео, новостные ленты).

**Ключевые моменты для запоминания из Лекции 8:**

*   **DSS (Системы поддержки принятия решений):** Помогают менеджерам решать сложные, часто полуструктурированные проблемы с помощью моделей и анализа "что-если". Два типа: Model-Driven и Data-Driven. Основные компоненты: СУБД, база моделей, пользовательский интерфейс.
*   **ES (Экспертные системы):** Используют ИИ для имитации мышления эксперта, предоставляя рекомендации на основе базы знаний (фактов и правил "ЕСЛИ...ТО...") и механизма вывода. Могут объяснять свою логику.
*   **KMS (Системы управления знаниями):** Помогают организациям создавать, хранить, делиться и использовать знания (явные и неявные), включая лучшие практики. Отличие данных, информации и знаний.
*   Все эти системы выходят за рамки традиционных функциональных систем и помогают принимать более качественные решения на разных уровнях организации.

# Lesson 9: Computer Systems (Hardware/Software), Telecommunications & The Internet

#### Часть 1: Компьютерные Компоненты и Функции Обработки (Слайды 2-58)

1.  **Введение в Компьютер (Слайд 3):**
    *   Компьютер – это информационный процессор, выполняющий арифметические и логические операции без вмешательства человека.
    *   **Базовая архитектура:** **ВВОД (INPUT) -> ЦЕНТРАЛЬНЫЙ ПРОЦЕССОР (CPU) + ОСНОВНАЯ ПАМЯТЬ (внутренняя) -> ВЫВОД (OUTPUT).** Также есть **ВСПОМОГАТЕЛЬНАЯ ПАМЯТЬ (внешняя)**.

2.  **Компоненты Компьютера (Слайды 4, 5):**
    *   **Центральный процессор (CPU - Central Processing Unit):** "Мозг" компьютера.
        *   **Арифметико-логическое устройство (АЛУ / ALU - Arithmetic/Logic Unit):** Выполняет математические расчеты и логические сравнения.
        *   **Устройство управления (Control Unit):** Последовательно получает инструкции программы, декодирует их и координирует поток данных между АЛУ, регистрами, основной и вторичной памятью, устройствами вывода.
        *   **Регистр (Register):** Маленькая ячейка памяти в CPU, где временно хранятся обрабатываемые инструкции.
    *   **Схема компонентов (Слайд 5):** Устройства ввода -> Процессор (УУ, АЛУ, Регистры, Основная память) -> Устройства вывода. Также есть связь с Вторичной памятью и Коммуникационными устройствами.

3.  **Характеристики и Функции Обработки (Слайд 6):**
    *   **Тактовая частота (Clock speed):** Скорость, с которой CPU выполняет операции. Измеряется в Мегагерцах (MHz – миллионы циклов в секунду) или Гигагерцах (GHz – миллиарды циклов в секунду).
    *   **Физические характеристики CPU:** Набор цифровых схем на кремниевых пластинах (чипах), размером не больше кончика карандаша.

4.  **Характеристики и Функции Памяти (Слайды 7, 8):**
    *   **Память (Memory):** Предоставляет CPU рабочее пространство для хранения программ и данных. Быстро передает данные и инструкции в CPU.
    *   **Емкость хранения (Storage capacity):** 8 бит = 1 Байт (Byte).
    *   **Ячейки основной памяти (Main Memory):** Хранят инструкции и данные для инструкций в электронной форме.
    *   **Инструкции для CPU:** Сообщают CPU, какие базовые операции выполнять (сложение, вычитание, умножение, деление, перемещение, хранение). Любая сложная задача разбивается на эти простые инструкции.

5.  **Как Компьютер Понимает Инструкции (Слайды 9-17):**
    *   **Системы счисления:**
        *   **Десятичная (Base 10):** Использует 10 цифр (0-9).
        *   **Двоичная (Base 2):** Использует 2 цифры (0 и 1). Компьютеры работают с двоичной системой. (Слайд 10)
    *   **Иерархия данных (Слайд 11):** От Бит/Байт (Символы) -> Поля -> Записи -> Файлы -> Базы Данных. (Повторение из Лекции 2)
    *   **Единицы цифрового измерения (Слайд 12, 13):**
        *   **Бит (Bit):** 0 или 1.
        *   **Байт (Byte):** 8 бит (обычно 1 символ).
        *   **Килобайт (KB):** ~1024 байта.
        *   **Мегабайт (MB):** ~1024 KB (миллион байт).
        *   **Гигабайт (GB):** ~1024 MB (миллиард байт).
        *   Далее: Терабайт (TB), Петабайт (PB), Эксабайт (EB), Зеттабайт (ZB), Йоттабайт (YB).
    *   **Машинный язык (Machine Language) (Слайд 14):** Инструкции на уровне 0 и 1. Самые первые программы писались на нем (язык первого поколения).
    *   **Системы кодирования символов (Слайд 14, 15):**
        *   Каждый символ на клавиатуре представляется последовательностью 0 и 1.
        *   **ASCII (American Standard Code for Information Interchange):** Распространенная схема кодирования (обычно 8 бит на символ).
        *   **EBCDIC (Extended Binary Coded Decimal Interchange Code):** Другая схема, использовалась в основном на мейнфреймах IBM.
        *   **Unicode:** Более современная схема (16 бит или больше), поддерживает символы многих языков мира.
    *   **Как компьютер использует двоичный код (Слайд 16):** Символ 'A' -> его двоичный код 01000001. Слово "HELLO" -> последовательность байтов для каждой буквы.
    *   **Язык Ассемблера (Assembly Language) – Второе поколение (Слайды 17, 18, 19, 31):**
        *   Использует символические обозначения (мнемоники, например, ADD, STO) вместо 0 и 1. Это упростило программирование.
        *   **Ассемблер (Assembler):** Программа-транслятор, которая переводит код на Ассемблере в машинный код.
        *   Одна инструкция на Ассемблере обычно транслируется в одну машинную инструкцию (низкоуровневый язык).
    *   **Языки высокого уровня (Higher Level Languages) – Третье поколение (3GL) (Слайды 20, 22-24, 32):**
        *   Не нужно думать на уровне машины.
        *   **Компилятор (Compiler) или Интерпретатор (Interpreter):** Транслируют код 3GL в машинный код.
        *   **Компилятор (Слайд 21):** Переводит всю программу (исходный код) в машинный код (объектный код) перед выполнением.
        *   В 3GL все еще нужно говорить компьютеру **ЧТО** делать и **КАК** это делать. Это **процедурные языки (PROCEDURAL Language)**.
        *   *Примеры 3GL:* COBOL (для бизнеса), FORTRAN (научный), BASIC, PASCAL, C, C++, C#, JAVA. Каждый имеет свою грамматику и подходит для разных задач.
    *   **Языки четвертого поколения (4GL) (Слайды 25, 26, 33):**
        *   Гораздо более дружелюбны к пользователю.
        *   Говорят компьютеру **ЧТО** делать, но **не КАК** это делать. Это **непроцедурные языки (NON-PROCEDURAL Language)**.
        *   *Пример:* команда "average <список чисел>".
        *   *Примеры 4GL:* DOS (команды), dBase (язык баз данных), SQL (язык запросов к БД), PowerBuilder.
        *   Обычные прикладные программы (текстовые процессоры, таблицы, браузеры) иногда относят к 4-му поколению ПО (но не языков программирования в строгом смысле), т.к. они непроцедурны и используют графический интерфейс (GUI).
    *   **Процедурное vs. Непроцедурное программирование (Слайд 27):**
        *   **Процедурное (3GL):** Нужно описать алгоритм (последовательность шагов) и закодировать его.
        *   **Непроцедурное (4GL):** Просто указать, что нужно сделать.
    *   **Пример: Нахождение среднего (Слайды 28-30):**
        *   **Алгоритм (Слайд 28):** Пошаговое описание, как найти среднее.
        *   **Кодирование алгоритма (Слайд 29):** Программа на процедурном языке (например, C) реализует этот алгоритм.
        *   **В непроцедурном языке (Слайд 30):** Просто команда "AVERAGE: числа" или ввод чисел в поле и нажатие кнопки "Average" в GUI.
    *   **Поколения языков программирования (Слайд 34):**
        *   **1940-е (1GL):** Машинный язык (двоичный).
        *   **1950-е (2GL):** Символический (Ассемблер).
        *   **Середина 1950-х (3GL):** Высокоуровневые (английские слова для процедур).
        *   **1970-е (4GL):** Ориентированные на результат (использование слов, сфокусированных на результате).
        *   **1990-е (5GL):** Искусственный интеллект, естественный язык (разговорный).

6.  **Аппаратное обеспечение компьютера (Hardware) (Слайды 35-46):**
    *   **Микрокомпьютер (ПК или Apple) (Слайд 36):**
        *   CPU – это микропроцессор (чип).
        *   Состоит из: системного блока, монитора, клавиатуры, мыши, принтера.
        *   **В системном блоке:** Материнская плата, дисководы, CD-ROM, платы, кабели, блок питания.
    *   **Материнская плата (Motherboard) (Слайды 37, 38):**
        *   Главная плата: микропроцессор (CPU), ОЗУ (RAM – оперативная память), шины, слоты для плат.
        *   Чипы Intel: 8088, ..., Pentium+++. Скорость в MHz или GHz.
    *   **RAM/ROM/Платы расширения (Слайд 39):**
        *   **RAM (Random Access Memory - Оперативная Память):** Основные чипы памяти (2-8 GB+). Хранит ОС, прикладное ПО, данные во время работы.
        *   **ROM (Read Only Memory - Постоянная Память):** "Прошитые" программы для запуска компьютера (например, BIOS).
        *   **Шины (Buses):** "Магистрали" для передачи инструкций между памятью и процессором.
        *   **Платы расширения (Expansion Cards):** Вставляются в слоты на материнской плате. Имеют **порты** для подключения периферийных устройств (принтеры, диски).
    *   **Хранение данных: Первичное/Вторичное (Слайд 40):**
        *   **Первичное (Внутреннее) хранилище (Primary Storage):** Основная память (RAM). Хранит инструкции и данные, обрабатываемые CPU. **Энергозависима** (содержимое стирается при отключении питания).
        *   **Вторичное (Внешнее) хранилище (Secondary Storage):** Устройства для долговременного хранения больших объемов данных, инструкций, информации. **Энергонезависима**. Большая емкость, экономичнее.
            *   *Распространенные виды:* Магнитные диски, ленты; Оптические носители; Твердотельные накопители.
    *   **Устройства вторичного хранения (Слайды 41-46):**
        *   **Последовательный доступ (Sequential Access) (Слайд 41):** Данные извлекаются в том порядке, в котором они были записаны (например, магнитная лента).
        *   **Прямой доступ (Direct Access) (Слайд 41):** Записи могут извлекаться в любом порядке (например, жесткий диск). Устройства прямого доступа – DASD.
        *   **Магнитный диск (Magnetic Disk) (Слайд 42):** Жесткий диск или дискета. Состоит из пластин с дорожками. Данные считываются/записываются головками. Позволяет прямой доступ.
        *   **Оптическое хранение (Optical Storage) (Слайд 43):** CD-ROM, DVD. Использует лазер. Может хранить больше данных на той же площади.
        *   **DVD (Digital Video Disc) (Слайд 44):** Для ПО, видеоигр, фильмов.
        *   **Твердотельные устройства вторичного хранения (Solid state secondary storage devices) (Слайд 44):** USB-флешки. Хранят данные на чипах памяти. Меньше движущихся частей, менее хрупкие. Дороже за ГБ, меньшая емкость по сравнению с HDD.
        *   **Сеть хранения данных (SAN - Storage Area Network) (Слайд 45):** Сеть, объединяющая устройства хранения, доступные нескольким компьютерам.
        *   **Хранение как услуга (Storage as a Service) (Слайд 46):** Провайдер сдает в аренду место для хранения данных, доступ через Интернет (облачное хранилище).

7.  **Вне системного блока (Слайд 47):**
    *   **Клавиатура:** При нажатии клавиши передается ее ASCII-код.
    *   **Монитор:** CRT (ЭЛТ), VDT (видеодисплейный терминал). Эволюция: CGA, EGA, VGA, Super VGA (больше цветов, точки ближе).
    *   **Принтер:** Ударный (матричный – старый) или безударный (струйный, лазерный).
    *   **Мышь:** Для использования с графическими интерфейсами (GUI).

8.  **Устройства ввода (Input Devices) (Слайды 48, 49):**
    *   Устройства ввода ПК, технология распознавания речи, цифровые камеры, сенсорные экраны, сканеры штрих-кодов, перьевые устройства ввода, карты с магнитной полосой, **RFID (Radio Frequency Identification)**.
    *   **RFID-метка (Слайд 49):** Маленькая метка, используемая для идентификации предметов (альтернатива штрих-кодам).

9.  **Устройства вывода (Output Devices) (Слайды 50, 51):**
    *   **Мониторы:**
        *   **Плазменные (Plasma display):** Тысячи "умных ячеек" (пикселей) с электродами и газами.
        *   **ЖК-дисплеи (LCD displays):** Плоские дисплеи на жидких кристаллах.
        *   **OLED (Organic light-emitting diodes):** Слой органического материала между двумя проводниками.
    *   **Принтеры и плоттеры:** Лазерные и струйные принтеры. Плоттеры – для чертежей. МФУ (многофункциональные устройства). 3D-принтеры.
    *   **Цифровой аудиоплеер:** Хранит, организует и воспроизводит музыку.
    *   **Электронные книги (E-books):** Цифровой эквивалент печатной книги.

10. **Типы Компьютерных Систем (Слайды 52-58):**
    *   От настольных/портативных до суперкомпьютеров.
    *   **Две основные группы компьютеров общего назначения (Слайд 52):**
        *   Однопользовательские (портативные и непортативные).
        *   Многопользовательские.
    *   **Портативные однопользовательские компьютеры (Слайд 53):**
        *   **Карманный компьютер (Handheld computer):** Компактное устройство с экраном (стилус/сенсорный), клавиатурой/цифровой панелью. Используются как POS-устройства. Есть защищенные версии для военных.
        *   **Ноутбуки (Laptop computers):** Для мобильных пользователей. Ноутбуки и ультрабуки меньше лэптопов. Планшеты – легкие, с клавиатурой или без.
    *   **Непортативные однопользовательские компьютеры (Слайды 54, 55):**
        *   **Тонкий клиент (Thin client):** Недорогой, централизованно управляемый компьютер без внутренних/внешних дисков для хранения.
        *   **Настольные компьютеры (Desktop computers):** Универсальные однопользовательские системы.
        *   **Неттоп (Nettop computer):** Недорогой настольный ПК, меньше, легче, потребляет меньше энергии.
        *   **Рабочие станции (Workstations):** Мощнее ПК, но достаточно малы для размещения на столе.
    *   **Многопользовательские компьютерные системы (Слайд 56):**
        *   **Сервер (Server):** Используется многими пользователями для выполнения конкретной задачи (сетевые или интернет-приложения).
        *   Серверные системы включают суперкомпьютеры, мейнфреймы и другие серверы.
        *   **Блейд-сервер (Blade server):** Сервер, в котором размещено много отдельных компьютерных материнских плат.
    *   **Классификация компьютеров (Слайд 57):**
        *   **Суперкомпьютеры:** Самые большие, мощные, дорогие; выполняют параллельную обработку (например, Big Blue от IBM).
        *   **Мейнфреймы (Mainframes):** Центральные, много "глупых" терминалов.
        *   **Миникомпьютеры (Minicomputers):** Меньшие мейнфреймы.
        *   **Микрокомпьютеры (Microcomputers):** Могут быть объединены в сеть; портативные, ноутбуки, планшеты и т.д.
        *   *Далее: Квантовые вычисления (Quantum Computing).*
    *   **Центры обработки данных (ЦОД / Data Centers) (Слайд 58):** Здание (или комплекс зданий) с контролируемым климатом и доступом, где размещается компьютерное оборудование, предоставляющее информационные услуги организации.

#### Часть 2: Программное обеспечение (Software) (Слайды 59-73)

1.  **Обзор Программного Обеспечения (Слайд 59):**
    *   **Компьютерные программы:** Последовательности инструкций для компьютера.
    *   **Документация:** Текст, описывающий функции программы для помощи пользователю.
    *   **Типы ПО:**
        *   **Системное ПО (Systems software).**
        *   **Прикладное ПО (Application software).**
    *   **Прикладное ПО:** Помогает пользователям решать конкретные проблемы. Обычно находится на жестком диске, может храниться на CD, DVD, USB.

2.  **Системное vs. Прикладное ПО (Слайд 60):**
    *   **Системное ПО:** Набор программ, координирующий работу аппаратного обеспечения и других программ. Разрабатывается для конкретного CPU и класса оборудования.
        *   *Компоненты (из схемы):* Инструменты управления файлами, Ассемблер, Отладчик, Операционная система, Утилиты, Компиляторы.
    *   **Прикладное ПО:** Помогает пользователям решать конкретные проблемы.
        *   *Примеры (из схемы):* Текстовый процессор, Электронные таблицы, Графика, Коммуникации, Базы данных, Игры.

3.  **Операционные Системы (ОС / Operating Systems) (Слайды 61-67):**
    *   **Определение (Слайд 61):** Набор программ, который управляет аппаратным обеспечением компьютера и служит интерфейсом с прикладными программами.
    *   **Задачи, выполняемые ОС (Слайд 62):**
        *   Выполнение общих функций аппаратного обеспечения.
        *   Предоставление пользовательского интерфейса и управление вводом/выводом.
        *   Обеспечение определенной степени независимости от оборудования.
        *   Управление системной памятью.
        *   Управление задачами обработки.
        *   Предоставление сетевых возможностей.
        *   Контроль доступа к системным ресурсам.
        *   Управление файлами.
    *   **Управление задачами обработки в ОС (Слайд 63):**
        *   **Многопользовательский режим (Multiuser):** Позволяет двум или более пользователям одновременно запускать программы на одном компьютере.
        *   **Многопроцессорность (Multiprocessing):** Поддерживает выполнение программы на более чем одном CPU.
        *   **Многозадачность (Multitasking):** Позволяет одновременно выполнять более одной программы.
        *   **Многопоточность (Multithreading):** Позволяет одновременно выполнять разные потоки одной программы. Поток – это независимый набор инструкций внутри приложения.
        *   **Реальное время (Real time):** Мгновенно реагирует на ввод.
    *   **Современные ОС (Слайды 64-67):**
        *   **Microsoft PC:** Windows (95, XP, Vista, 7, 8, 10). (Слайд 64)
        *   **Apple:** Mac OS X (Leopard, Snow Leopard, Lion, ..., Sierra). (Слайд 65)
        *   **Linux:** ОС с открытым исходным кодом. Red Hat – самый распространенный дистрибутив. (Слайд 66)
        *   **Chrome OS:** ОС на базе Linux для нетбуков и неттопов. Chromium OS – версия с открытым исходным кодом.
        *   **Android:** ОС для мобильных устройств. (Слайд 66)
        *   **Мобильные ОС (статистика за 2 кв. 2013 г. – Слайд 67):** Google Android (лидер), Apple iPhone OS, Microsoft Windows Mobile, Blackberry.

4.  **Утилиты Операционных Систем (Utilities) (Слайды 68, 69):**
    *   Программы, управляющие компьютерными ресурсами и файлами. Могут быть частью ОС или приобретаться отдельно.
    *   Помогают выполнять обслуживание или исправлять проблемы. Обеспечивают безопасность данных.
    *   **Примеры утилит (Слайд 68):** Резервное копирование (Backup), Дефрагментация диска, Восстановление данных, Сжатие данных, Конвертация файлов, Антивирус, Драйверы устройств, Блокировщики спама, Обнаружение шпионского ПО, Медиаплееры.
    *   **Уровни утилит (Слайд 69):** Персональные (сжатие, очистка диска, антивирус), Рабочая группа (архивирование изменений, мониторинг активности группы, отчеты о неудачных входах), Корпоративные (архивирование БД, мониторинг трафика, статус заданий).

5.  **Прикладное Программное Обеспечение (Application Software) (Слайды 70-73):**
    *   Программы, которые взаимодействуют с системным ПО, а системное ПО управляет аппаратной частью для выполнения задач. (Слайд 70)
    *   Помогают выполнять общие задачи: создание документов, вычисления, управление информацией. Некоторые приложения более специализированы.
    *   **Типы прикладного ПО (Слайд 71):**
        *   **Заказное ПО (Proprietary software):** Уникальная программа для конкретного приложения, обычно разрабатывается и принадлежит одной компании.
        *   **Готовое ПО (Off-the-shelf software / COTS - Commercial Off The Shelf):** Существующая программа, которую покупают.
        *   **Поставщик услуг приложений (ASP - Application service provider):** Компания, предоставляющая ПО, поддержку и оборудование для запуска ПО с мощностей пользователя через сеть.
    *   **Сравнение заказного и готового ПО (Слайд 72):** (Подробная таблица преимуществ и недостатков каждого).
        *   *Заказное:* Преимущества – точно под нужды, контроль над разработкой, возможность модификации. Недостатки – долго и дорого разрабатывать, сложность поддержки.
        *   *Готовое:* Преимущества – низкая начальная стоимость, быстрая доступность, проверенное качество. Недостатки – может не полностью соответствовать нуждам, плата за ненужные функции, сложность кастомизации.
    *   **ПО как услуга (SaaS - Software as a service) (Слайд 73):**
        *   Позволяет бизнесу подписываться на бизнес-приложения, доставляемые через веб, с ежемесячной оплатой или оплатой за использование.
        *   Может снизить расходы за счет совместного использования приложений многими компаниями.
    *   **Облачные вычисления (Cloud computing) (Слайд 73):**
        *   Использование вычислительных ресурсов (включая ПО и хранение данных) в Интернете (в "облаке"), а не на локальных компьютерах.

#### Часть 3: Компьютерные Сети, Телекоммуникации и Интернет (Слайды 74-130)

*(Эта часть очень большая, охватывает много тем)*

1.  **Компьютерные Сети (Computer Networking) (Слайды 74, 76):**
    *   Сети существовали давно (например, водопровод в Древнем Риме).
    *   **Сеть:** Группа взаимосвязанных людей или вещей.
    *   **Компьютерная сеть:** Группа из двух или более вычислительных устройств, связанных какой-либо средой передачи.
    *   Используют вычислительные и телекоммуникационные технологии.
    *   **Зачем нужны сети?** Для обмена электронной информацией, повышения эффективности, совместного использования оборудования, программ и баз данных.
    *   **Сетевые узлы (Network nodes):** Компьютеры и устройства в сети.

2.  **Телекоммуникации (Telecommunications) (Слайд 75):**
    *   Электронная передача сигналов для связи.
    *   **Телекоммуникационная среда (Telecommunications medium):** Любое материальное вещество, передающее электронный сигнал между отправляющим и принимающим устройством.

3.  **Топология Сети (Network Topology) (Слайд 77):** Как организованы каналы связи и аппаратные устройства в сети.
    *   **Звезда (Star Network):** Устройства подключены к центральному узлу (хабу).
    *   **Шина (Bus Network):** Есть общая магистраль (backbone).
    *   **Ячеистая (Mesh Network):** Все устройства являются точками доступа и соединены друг с другом (или многие со многими).

4.  **Клиент-Серверные Системы (Client/Server Systems) (Слайд 78):**
    *   Архитектура с несколькими компьютерными платформами, выделенными под специальные функции (управление БД, печать, коммуникации).
    *   **Клиент (Client):** Любой компьютер, отправляющий сообщения с запросом услуг от серверов в сети.
    *   Сервер базы данных отправляет только те данные, которые соответствуют запросу, а не весь файл.

5.  **Типы Сетей (Network Types) (Слайд 79):**
    *   **Персональные сети (PAN - Personal Area Networks):** Связь устройств в радиусе ~10 метров (33 фута).
    *   **Локальные сети (LAN - Local Area Networks):** Соединяют компьютеры и устройства на небольшой территории (офис, дом). Частные кабели (витая пара, коаксиал). Могут быть беспроводными. Обычно есть файловый сервер.
    *   **Глобальные сети (WAN - Wide Area Networks):** Соединяют большие географические регионы. Состоят из оборудования пользователя и линий связи, предоставляемых операторами.

6.  **Скорость Передачи (Speed of Transmission) (Слайд 80):**
    *   **Пропускная способность канала (Channel bandwidth):** Скорость обмена данными, измеряется в битах в секунду (бит/с).
    *   **Широкополосная связь (Broadband communications):** Относительный термин; система, передающая данные очень быстро (> 25 Мбит/с).
    *   Единицы измерения: бит/с (bps), Кбит/с (Kbps – тысячи), Мбит/с (Mbps – миллионы), Гбит/с (Gbps – миллиарды).

7.  **Среды Передачи (Transmission Media) (Слайды 81-91):**
    *   Физические пути между участниками сети.
    *   **Две широкие категории (Слайд 81):**
        *   **Проводные (Wired / guided):** Сигналы направляются по твердой среде. Кабели: витая пара, коаксиальный, оптоволоконный.
        *   **Беспроводные (Wireless):** Сигнал транслируется по воздуху в виде электромагнитного излучения. Инфракрасная линия видимости, высокочастотное радио, микроволны, Wi-Fi.
    *   **Проводные среды (Слайд 82):**
        *   **Витая пара (Twisted-pair wire):** Экранированная или неэкранированная. Для телефонной связи. Ограничения по передаче и расстоянию.
        *   **Коаксиальный кабель (Coaxial Cable):** Внутренний проводник, окруженный изоляцией. Чище и быстрее витой пары, но дороже.
        *   **Оптоволокно (Fiber Optic):** Тонкие стеклянные нити, используют световые лучи. Меньший диаметр, меньше искажений сигнала, высокие скорости передачи. Еще дороже.
    *   **Беспроводные технологии (Слайды 83-91):**
        *   **Частотные диапазоны (Слайд 83):** Радио, микроволны, инфракрасные. Использование может регулироваться для избежания помех.
        *   **Сотовая связь (Cellular Phone) (Слайд 84):** Двусторонняя беспроводная связь, назначает уникальные частоты звонкам.
        *   **Смартфоны (Слайд 85):** Комбинируют функции телефона, камеры, браузера и т.д. Имеют свои ОС.
        *   **Поколения беспроводной связи (3G, 4G, 5G) (Слайд 86):** Каждое следующее поколение предлагает более высокие скорости передачи данных. LTE (Long Term Evolution) – часть 4G. 5G – новейшее поколение.
        *   **Другие беспроводные технологии (Слайд 87):**
            *   **NFC (Near Field Communication):** Беспроводная связь малого радиуса действия (телефоны, кредитные карты).
            *   **Bluetooth:** Расстояние 10-30 футов, скорость ~2 Мбит/с. Устройства должны быть сопряжены.
            *   **Wi-Fi:** Связь через беспроводной адаптер на устройстве. Адаптер переводит данные в радиосигналы. Точка беспроводного доступа подключена к компьютеру.
        *   **Беспроводные среды (Слайд 88):**
            *   **Инфракрасная линия видимости (Infrared Line of Site - LOS):** Как пульт от телевизора.
            *   **Высокочастотное радио:** Нужны антенные вышки (пейджеры, сотовые, рации).
            *   **Микроволны (Microwave):** Большие расстояния.
                *   **Наземные (Terrestrial) (Слайд 89):** Антенны каждые ~30 миль. Технология прямой видимости.
                *   **Спутниковые (Satellite) (Слайд 90):** Сигналы от земных антенн к спутникам и обратно. Для удаленных мест.

8.  **Интернет – "Сеть сетей" (The Internet) (Слайды 92-100):**
    *   Огромная всемирная совокупность сетей, использующих общий протокол для связи. (Слайд 92)
    *   **История (Слайд 93):**
        *   **ARPANET (1960-е, DARPA):** Для связи правительства и университетов в исследовательских целях.
        *   **NSFNET (1986, National Science Foundation):** Для соединения исследовательских институтов.
        *   Соединились с ARPANET и другими, став основой Интернета.
        *   Поддержка от университетов, правительств, исследовательских институтов, индустрии.
    *   **Как работает Интернет – Подключение (Слайд 94):**
        *   **Модем (Modulate/Demodulate):** Преобразует сигналы из цифрового в аналоговый и обратно.
        *   **Интернет-провайдер (ISP - Internet Service Provider):** Компании, предоставляющие доступ в Интернет за плату (например, MTS, Shaw).
        *   **Точки доступа к сети (NAP - Network Access Points):** Соединяют ISP между собой. Служат точками обмена интернет-трафиком.
        *   **Магистраль Интернета (Internet Backbone):** Совокупность основных сетевых соединений и телекоммуникационных линий, составляющих Интернет. (Слайд 95)
    *   **Сетевые протоколы (Network Protocols) (Слайд 96):**
        *   **Протокол:** Набор правил о том, КАК что-то делать.
        *   В сетях: согласованные форматы для передачи данных, как формировать пакеты, сигнализировать конец сообщения, указывать адрес назначения.
    *   **TCP/IP и Маршрутизаторы (Routers) (Слайд 97):**
        *   **TCP (Transmission Control Protocol):** Разбивает информацию на пакеты данных, управляет передачей пакетов, собирает пакеты в сообщение у получателя.
        *   **IP (Internet Protocol):** Контролирует формирование пакетов данных, адресует каждый пакет (источник и назначение). Пакет данных IP называется **IP-датаграммой**.
        *   **Маршрутизаторы (Routers):** Соединяют одну сеть с другой. Идентифицируют каждое устройство в сети по IP-протоколу. Служат "регулировщиками трафика", направляя пакеты к месту назначения.
    *   **Коммутация пакетов (Packet Switching) (Слайды 98, 99, 100):**
        *   Интернет использует коммутацию пакетов для одновременной передачи данных многими пользователями.
        *   Данные "нарезаются" на пакеты.
        *   Каждый пакет имеет заголовок с адресом источника и назначения.
        *   Каждый компьютер в сети имеет уникальный сетевой адрес.
        *   Пакеты могут идти разными путями и собираются в пункте назначения. (Схема на Слайде 99).

9.  **Всемирная паутина (World Wide Web - WWW) (Слайды 101-114):**
    *   Разработана Тимом Бернерсом-Ли в ЦЕРНе (CERN) около 1980 г. Изначально как внутренняя система управления документами. (Слайд 101)
    *   Стала: основным источником новостей и информации, незаменимым каналом для коммерции, популярным центром для социального взаимодействия, развлечений и общения.
    *   **Веб-браузеры (Web Browsers) (Слайд 102):** Программы для поиска и отображения веб-страниц (текст, графика, мультимедиа). (Примеры: Safari, Chrome, Firefox, Opera, Edge).
    *   **Архитектура WWW (Слайд 103):** Интернет-пользователь через браузер обращается к веб-серверам, на которых хранятся веб-сайты (набор веб-страниц, связанных гиперссылками).
    *   **IP-адреса (Internet Protocol Addresses) (Слайд 104):**
        *   Каждое доменное имя (например, uwinnipeg.ca) связано с IP-адресом.
        *   Номер, присвоенный каждому устройству в сети, использующей IP.
        *   **IPv4:** 32-битный числовой адрес (4.29 млрд адресов), записывается как 4 числа, разделенных точками (например, 1.160.10.240).
        *   Доменное имя преобразуется в IP-адрес специальным сервером (DNS - Domain Name System).
    *   **IPv6 (Слайд 105):** Запущен в 2012 г. 128-битные адреса (квадриллионы адресов). Записывается как 6 групп шестнадцатеричных чисел, разделенных двоеточиями.
    *   **Веб-серверы (Web Servers) (Слайд 106):** Специальные компьютеры для хранения и "раздачи" веб-страниц. Имеют специальное аппаратное и программное обеспечение.
    *   **Ключевые понятия WWW (Слайд 107):**
        *   **Гипертекст (Hypertext):** Веб-страница, хранящаяся на веб-сервере. Содержит информацию и ссылки (гиперссылки) на другую связанную информацию.
        *   **HTML (Hypertext Markup Language):** Стандартный метод для указания формата веб-страниц. Использует коды/теги для отображения контента. (Пример HTML-кода на Слайде 110).
        *   **Веб-браузер:** Программа для поиска и отображения веб-страниц.
    *   **Веб-адреса и Домены (Слайд 108):**
        *   **Домен (Domain):** Идентифицирует веб-сайт (хост). Суффиксы: .edu (образовательные), .org (некоммерческие), .mil (военные), .net (сетевые организации), .com (коммерческие), .ca (Канада) и т.д. (Слайды 112, 113).
        *   **URL (Uniform Resource Locator):** Идентифицирует конкретные веб-страницы внутри домена.
    *   **Веб-сайт (Website) (Слайд 111):** Коллекция взаимосвязанных веб-страниц, созданных одним автором (авторами) для общей цели. Отвечает на запросы из браузеров по протоколу HTTP.
    *   **Типы веб-сайтов (Слайд 114):**
        *   **Статические:** Коллекция статических HTML-документов, связанных ссылками.
        *   **Статические с формами:** В основном доставка документов, но есть формы для сбора информации от пользователя.
        *   **Динамический доступ к данным:** Через веб-страницу пользователи могут искать в каталоге или выполнять запросы к базе данных (например, регистрация на курсы).
        *   **Веб-приложения:** Облегчают бизнес-процессы, выходящие за рамки простого информирования; бизнес-ИС на веб-сайте (например, отслеживание запасов, автоматизация продаж).

10. **Облачные Вычисления (Cloud Computing) (Слайды 115, 116):**
    *   ПО и хранилище, предоставляемые как интернет-сервис и доступные через веб-браузер. (Слайд 115)
    *   *Примеры:* Email, хранение данных (Skydrive), налоговое ПО, Flickr, Facebook.
    *   Скоро большинство вычислений будет происходить в интернете.
    *   **Преимущества для бизнеса (Слайд 116):**
        *   Аутсорсинг ресурсов – экономия на проектировании, установке, обслуживании систем.
        *   Доступ к корпоративным системам с любого устройства, подключенного к интернету.
        *   Увеличение возможностей хранения данных.
        *   Ответственность за сохранность данных ложится на поставщика услуг.
    *   *Пример: Облачный центр Google (Слайд 117).*

11. **Поисковые Системы (Search Engines) (Слайды 118, 119):**
    *   Программы, которые ищут документы по указанным ключевым словам и возвращают список документов (веб-страниц), где эти слова найдены. (Слайд 118)
    *   Крупнейшее приложение в вебе.
    *   Прибыльный бизнес (Google).
    *   Важный вклад в развитие Веба и Интернета.
    *   **SEO (Search Engine Optimization):** Оптимизация веб-сайтов для поисковых систем.
    *   **Как работает Google (Слайд 119):** Пользователь вводит запрос -> Серверы Google получают запрос -> Запрос отправляется на индексные серверы Google (хранят данные о веб-страницах) -> ПО PageRank оценивает "важность" страниц -> Готовятся краткие описания страниц -> Результаты доставляются пользователю.

12. **Обзор Веб-сессии (Слайды 120-122):**
    1.  Пользователь (в браузере) запрашивает документ с удаленного веб-сервера, вводя URL.
    2.  Фактический IP-адрес определяется с помощью DNS-сервера.
    3.  Как только машина найдена, запрос отправляется на сервер. Обмен данными между браузером и веб-сервером происходит по протоколу **HTTP (Hypertext Transfer Protocol)** – определяет, как клиент запрашивает данные и как сервер отвечает. (HTTP не определяет, как данные передаются, это делает TCP/IP на более низком уровне).
    4.  Сервер пытается обработать запрос (проверяет валидность). Отправляет данные с диска в сеть (используя ПО веб-сервера, например Apache).
    5.  Браузер на клиенте читает полученные данные, идентифицирует их как HTML-документ и отображает страницу в окне браузера.

13. **Итог по Интернету (Слайд 123): Три важные концепции:**
    1.  Клиент-серверные вычисления.
    2.  Коммутация пакетов.
    3.  Протокол связи TCP/IP.

14. **Эволюция Веба (Слайды 124, 125, 126):**
    *   **Web 1.0 ("Бизнес Веб"):** Отображение простых страниц, нелинейная навигация. (1994-2000)
    *   **Web 2.0 ("Социальный Веб"):** Пользователь-ориентированные приложения и технологии социальных медиа. Пользовательский контент, интерактивность, социальные сообщества. (Примеры: Twitter, YouTube, Instagram, Wikipedia). (2001-настоящее время, с периодом консолидации после краха доткомов).
    *   **Web 3.0 ("Семантический Веб" / "Интеллектуальный Веб" / "Интернет Вещей"):** Конвергенция технологий: повсеместная связь (мобильные устройства), сетевые вычисления (SaaS), открытые технологии (API, открытые данные, open source), открытая идентичность (портативная идентичность и личные данные).

15. **Будущее Интернета (Слайды 127-130):**
    *   **Проект Internet2 (Слайд 127):** Консорциум из 350+ учреждений, сотрудничающих для содействия революционным интернет-технологиям. Гарантированные уровни обслуживания, низкие показатели ошибок, возможность покупать право на передачу данных с гарантированной скоростью за более высокую плату, снижение затрат.
    *   **Интернет Вещей (IoT - Internet of Things) (Слайды 127, 128, 129, 130):**
        *   Сеть физических объектов ("вещей"), оснащенных датчиками, процессорами, ПО и сетевыми возможностями для обмена данными с производителем устройства, операторами и другими подключенными устройствами.
        *   **Датчик (Sensor):** Устройство, способное воспринимать что-либо об окружающей среде (давление, температура, влажность, движение и т.д.).
        *   *Процесс IoT (Слайд 129):* Датчики собирают данные -> Данные передаются по сети -> Данные собираются и хранятся (часто в облаке) -> Данные объединяются с данными из других систем -> Данные анализируются для получения инсайтов -> Оповещения отправляются людям, корпоративным системам или устройствам IoT на основе этих инсайтов.
        *   *Примеры использования датчиков и IoT (Слайд 130):* Мониторинг активов, строительство, сельское хозяйство, производство, мониторинг парковочных мест, предиктивное обслуживание, розничная торговля, мониторинг трафика.

# Lesson 10: eBusiness/eCommerce Systems

1.  **Введение в Электронный Бизнес и Электронную Коммерцию (Слайды 2, 3):**
    *   В 1994 году электронная коммерция в ее нынешнем виде **еще не существовала**.
    *   **Объемы в 2018 году:**
        *   Покупка товаров, услуг и цифрового контента потребителями через ПК или мобильные устройства: **$1 трлн** (190 млн американских потребителей).
        *   Затраты бизнеса на закупки через интернет: **$6.1 трлн**.
    *   **Ключевые моменты в истории:**
        *   **2000-2001:** Крах фондового рынка (пузырь доткомов).
        *   **2002-2007:** Розничная электронная коммерция росла более чем на 25% в год.
        *   **2007:** Apple представила первый iPhone – **новая эра в электронной коммерции** (мобильная коммерция).
    *   **Влияние мобильных устройств:** За последние 10 лет смартфоны, планшеты и мобильные приложения вытеснили традиционные ПК/ноутбуки и веб-браузеры как основной способ доступа потребителей к интернету. Технологии (сотовые сети, Wi-Fi, облачные вычисления) превратили мобильные устройства в инструменты для рекламы, покупок, чтения и просмотра медиа, снова изменив поведение потребителей.

2.  **Влияние Электронной Коммерции на Общество (Слайд 4):**
    *   **Позитивное влияние:** Рост, инновации, создание рабочих мест.
    *   **Проблемы (Challenges):**
        *   Вторжение в частную жизнь (Privacy invasion).
        *   Распространение дезинформации.
        *   Угрозы безопасности.
        *   Доминирование на рынке технологических гигантов (Amazon, Google, Facebook).
    *   **Регулирование (Regulation):** Усиление надзора за цифровым бизнесом.

3.  **Будущее Электронной Коммерции (Слайд 5):**
    *   **Продолжающийся рост:** Прогноз на 2022 год: потребительские расходы – $1.5 трлн, расходы бизнеса – $7.3 трлн.
    *   **Технологические достижения:** Искусственный интеллект (AI), облачные вычисления, блокчейн, эволюция мобильных и цифровых платформ.
    *   **Доминирование электронной коммерции:** Ожидается, что к 2050 году большая часть коммерции будет цифровой.

4.  **Зачем Изучать Электронную Коммерцию (Слайд 6):**
    *   Понимание **трендов и бизнес-моделей**.
    *   Изучение **технологических основ** (Интернет, Веб, мобильные платформы).
    *   Рассмотрение вопросов **безопасности и этики**.
    *   Изучение **маркетинговых стратегий** (социальный, мобильный, локальный маркетинг).

5.  **Что такое Электронная Коммерция (E-Commerce)? (Слайд 7):**
    *   Использование Интернета, Всемирной паутины (Web) и мобильных приложений/браузеров для ведения бизнеса.
    *   **Определение:** Коммерческие транзакции между организациями и частными лицами, осуществляемые с помощью цифровых технологий.
    *   **Включает:**
        *   **Интернет:** Глобальная сеть компьютерных сетей.
        *   **Веб (Web):** Основной сервис Интернета, предоставляющий веб-страницы.
        *   **Мобильные приложения и браузеры:** Позволяют совершать транзакции на мобильных устройствах.
    *   **Обмен ценностями (Exchange of Value):** Ключевой элемент для любой коммерции.

6.  **Ключевые Компоненты Электронной Коммерции (Слайд 8):**
    *   **Транзакции, осуществляемые с помощью цифровых технологий (Digitally Enabled Transactions):** Проводятся через Интернет, Веб или мобильные устройства.
    *   **Коммерческие транзакции (Commercial Transactions):** Обмен деньгами, продуктами или услугами.
    *   **Электронная коммерция (E-Commerce) vs. Цифровая коммерция (Digital Commerce):** Используются взаимозаменяемо, означают одно и то же.

7.  **Электронная Коммерция (E-Commerce) vs. Электронный Бизнес (E-Business) (Слайды 9, 10):**
    *   **E-Commerce:**
        *   Цифровые транзакции, включающие **внешних клиентов или партнеров**.
        *   *Пример:* Онлайн-покупки, цифровые платежи.
    *   **E-Business:**
        *   **Внутренние** цифровые процессы, поддерживающие бизнес-функции.
        *   *Пример:* Управление запасами, внутренние системы данных.
    *   **Ключевое различие:** E-Commerce включает **обмен ценностями через организационные границы**. E-Business – это применение цифровых технологий для бизнес-процессов *внутри* фирмы. (Схема на слайде 10 иллюстрирует это: e-business – это более широкое понятие, включающее внутренние процессы, а e-commerce – это часть, связанная с внешними транзакциями с клиентами и поставщиками).

8.  **Основные Тренды в Электронной Коммерции (Слайды 11, 12, 13):**
    *   **Бизнес-тренды (Слайд 11):**
        *   Розничная e-commerce в США продолжает расти двузначными темпами (>15%), на развивающихся рынках (Азиатско-Тихоокеанский регион, Ближний Восток, Африка) темпы еще выше.
        *   Мобильная e-commerce (розница и путешествия) стремительно растет.
        *   Рынок мобильных приложений продолжает расти, миллионы американцев используют смартфоны и планшеты.
        *   Социальная e-commerce (на базе соцсетей и рекламы) растет.
        *   Локальная e-commerce (третье измерение – мобильная, социальная, локальная) также растет, стимулируемая сервисами по требованию (типа Uber).
        *   B2B e-commerce достигает триллионных оборотов.
        *   Сервисы по требованию (Uber, Airbnb) привлекают миллиарды инвестиций.
        *   Мобильная реклама растет астрономическими темпами, составляя >70% всех цифровых рекламных расходов.
        *   Малый бизнес и предприниматели продолжают выходить на рынок e-commerce, часто используя инфраструктуру гигантов (Apple, Facebook, Amazon, Google, eBay).
    *   **Технологические тренды (Слайд 12):**
        *   Мобильная платформа (смартфоны, планшеты, носимые устройства, мобильные приложения) становится альтернативной платформой для онлайн-транзакций, маркетинга, рекламы, просмотра медиа. Растет использование мобильных мессенджеров (Facebook Messenger, WhatsApp, Snapchat).
        *   Умные колонки (Amazon Echo, Google Home) становятся все популярнее, предоставляя дополнительную платформу для e-commerce.
        *   Облачные вычисления завершают трансформацию мобильной платформы, храня контент и ПО в "облаке" и делая их доступными с любого устройства.
        *   Интернет вещей (IoT) – миллиарды подключенных к интернету устройств – продолжает экспоненциально расти.
        *   Компании отслеживают триллионы онлайн-взаимодействий, генерируя огромные объемы данных (Big Data).
        *   Для осмысления Big Data используется сложная бизнес-аналитика (или веб-аналитика), выявляющая модели покупок и интересы потребителей за миллисекунды.
    *   **Социальные тренды (Слайд 13):**
        *   Пользовательский контент (посты в соцсетях, твиты, блоги) растет и предоставляет метод самопубликации, привлекающий миллионы.
        *   Соцсети поощряют самораскрытие, но угрожают конфиденциальности (пример: Facebook и Cambridge Analytica).
        *   Растут опасения по поводу доминирования на рынке Facebook, Amazon, Google, что приводит к призывам к государственному регулированию.
        *   Продолжаются конфликты по управлению авторскими правами, но есть согласие между онлайн-дистрибьюторами и владельцами контента, что они нужны друг другу.
        *   Верховный суд США постановил, что онлайн-бизнес должен собирать налог с продаж штата.
        *   Растет слежка за онлайн-коммуникациями со стороны репрессивных режимов и западных демократий.
        *   Растут опасения по поводу коммерческого и государственного вторжения в частную жизнь.
        *   Онлайн-безопасность продолжает снижаться из-за взломов крупных компаний и потери контроля над информацией клиентов.
        *   Спам остается серьезной проблемой, несмотря на законодательство и технологии.
        *   E-commerce по требованию порождает массу временных, низкооплачиваемых рабочих мест без льгот.

9.  **Восемь Уникальных Особенностей Технологии E-Commerce и их Значение для Бизнеса (Слайд 14):**
    *(Очень важный слайд, часто встречается в учебниках по e-commerce)*
    *   **Вездесущность (Ubiquity):** Технология доступна везде: на работе, дома, в пути через мобильные устройства, в любое время.
        *   *Значение:* Рынок выходит за традиционные границы, не привязан к времени и месту. "Рыночное пространство" (Marketspace) может быть где угодно. Удобство для клиента повышается, затраты на поиск снижаются.
    *   **Глобальный охват (Global reach):** Технология охватывает национальные границы по всему миру.
        *   *Значение:* Коммерция становится возможной через культурные и национальные границы без особых модификаций. "Рыночное пространство" включает потенциально миллиарды потребителей и миллионы бизнесов по всему миру.
    *   **Универсальные стандарты (Universal standards):** Существует единый набор технологических стандартов.
        *   *Значение:* Единая, недорогая, глобальная технологическая основа для использования бизнесом.
    *   **Богатство (Richness):** Возможны видео-, аудио- и текстовые сообщения.
        *   *Значение:* Видео-, аудио- и текстовые сообщения интегрируются в единый маркетинговый посыл и потребительский опыт.
    *   **Интерактивность (Interactivity):** Технология работает через взаимодействие с пользователем.
        *   *Значение:* Потребители вовлекаются в диалог, который динамически подстраивает опыт под индивидуума и делает потребителя соучастником процесса доставки товаров на рынок.
    *   **Информационная плотность (Information density):** Технология снижает информационные затраты и повышает качество.
        *   *Значение:* Затраты на обработку, хранение и передачу информации снижаются, а актуальность, точность и своевременность значительно улучшаются. Информация становится обильной, дешевой и точной.
    *   **Персонализация/Кастомизация (Personalization/Customization):** Технология позволяет доставлять персонализированные сообщения отдельным лицам и группам.
        *   *Значение:* Позволяет персонализировать маркетинговые сообщения и кастомизировать продукты и услуги на основе индивидуальных характеристик.
    *   **Социальные технологии (Social technology):** Пользовательский контент и социальные сети.
        *   *Значение:* Обеспечивает создание пользовательского контента и дистрибуцию, поддерживает развитие социальных сетей.

10. **Типы Электронной Коммерции (Слайд 15):**
    Различаются по характеру рыночных отношений – **кто кому продает**.
    *   **B2C (Business-to-Consumer):** Бизнес – Потребителю.
    *   **B2B (Business-to-Business):** Бизнес – Бизнесу.
    *   **C2C (Consumer-to-Consumer):** Потребитель – Потребителю.
    *   **C2B (Consumer-to-Business):** Потребитель – Бизнесу.
    *   **G2B & G2C (Government models):** Правительство – Бизнесу / Правительство – Потребителю.
    *   Мобильная, социальная и локальная e-commerce рассматриваются как подмножества этих типов.

11. **B2C (Бизнес – Потребителю) (Слайд 16):**
    *   Самый распространенный тип. Бизнес продает напрямую индивидуальным потребителям.
    *   Включает розничные товары, путешествия, финансовые услуги, недвижимость, онлайн-контент.
    *   Экспоненциальный рост с 1995 года, ожидается дальнейший рост.
    *   **Семь основных моделей B2C:**
        *   **Онлайн-ритейлеры (Online retailers):** Amazon, Walmart.com.
        *   **Поставщики услуг (Service providers):** Netflix, Uber.
        *   **Транзакционные брокеры (Transaction brokers):** Expedia, PayPal.
        *   **Поставщики контента (Content providers):** YouTube, Spotify.
        *   **Поставщики сообществ/Социальные сети (Community providers/Social networks):** Facebook, Twitter.
        *   **Создатели рынка (Market creators):** eBay, Airbnb.
        *   **Порталы (Portals):** Yahoo, Google News.

12. **B2B (Бизнес – Бизнесу) (Слайд 17):**
    *   **Крупнейшая форма e-commerce** (по объему транзакций).
    *   Компании продают другим компаниям (не индивидуальным потребителям).
    *   Составляет значительную часть мировой торговли.
    *   Огромный потенциал роста по мере внедрения онлайн-закупок бизнесом.

13. **C2C (Потребитель – Потребителю) (Слайд 18):**
    *   Потребители продают друг другу с помощью **онлайн-маркетплейса** (поставщика платформы).
    *   Потребитель готовит товар и выставляет его на аукцион или продажу, а маркетплейс предоставляет каталог, поиск и возможности транзакций.
    *   **Поставщики платформ:** eBay, Craigslist, Etsy, продажи через сторонних продавцов на Amazon, Facebook Marketplace и др.
    *   Значительный размер рынка.

14. **C2B (Потребитель – Бизнесу) (Слайд 19):**
    *   Потребители предлагают товары или услуги бизнесу.
    *   **Распространенные платформы:** Фриланс-платформы (Fiverr, Upwork).
    *   **Характеристики:** Бизнес покупает услуги или продукты у частных лиц. Фрилансеры или индивидуальные предприниматели предоставляют услуги (дизайн, написание текстов, консалтинг).
    *   **Рост:** С развитием "гиг-экономики" (экономики временных подработок) C2B-платформы становятся все популярнее для доступа бизнеса к талантам по требованию.

15. **G2B (Правительство – Бизнесу) и G2C (Правительство – Потребителю) (Слайд 20):**
    *   **G2B:** Правительства предлагают онлайн-услуги бизнесу.
        *   *Примеры:* Онлайн-подача налоговых деклараций, получение бизнес-лицензий, цифровые государственные услуги.
    *   **G2C:** Правительства предлагают цифровые услуги напрямую гражданам.
        *   *Примеры:* Онлайн-подача налогов, регистрация в системе здравоохранения, государственные пособия.
    *   **Особенности:** Оптимизированные процессы взаимодействия граждан и бизнеса с госуслугами; повышение эффективности за счет цифровых инструментов и платформ.

16. **Мобильная E-commerce (M-commerce) (Слайд 21):**
    *   Онлайн-транзакции через мобильные устройства (смартфоны, планшеты).
    *   **Рост:** Значительный объем и прогнозируемый дальнейший рост.
    *   **Ключевые драйверы:**
        *   Рост использования мобильных устройств и увеличение размеров экранов.
        *   Сайты, оптимизированные для мобильных устройств, и улучшенные мобильные платежи.
        *   Рост **разговорной коммерции (conversational commerce)** через чат-ботов в приложениях (Facebook Messenger, WhatsApp).

17. **Социальная E-commerce (Social e-commerce) (Слайд 22):**
    *   E-commerce, основанная на социальных сетях и онлайн-отношениях.
    *   **Драйверы роста:**
        *   **Вход через соцсети (Social sign-on):** Авторизация с помощью аккаунтов соцсетей.
        *   **Уведомления в сети:** Распространение одобрения продуктов/услуг.
        *   **Инструменты для совместных покупок и социальный поиск.**
        *   Интегрированные инструменты: кнопки "Купить", вкладки "Магазин", группы-маркетплейсы на платформах (Facebook, Instagram, Pinterest, YouTube).
    *   **Рост:** Значительный рост объемов.

18. **Локальная E-commerce (Local e-commerce) (Слайд 23):**
    *   E-commerce, нацеленная на потребителей на основе их **географического положения**.
    *   **Ключевые особенности:**
        *   Местные продавцы используют онлайн-маркетинг для привлечения посетителей в обычные магазины.
        *   Часто интегрирована с **сервисами по требованию (on-demand services)** (например, Uber).
    *   **Рост:** Значительные объемы в США.

**Ключевые моменты для запоминания из Лекции 10:**

*   **Разница между E-Commerce и E-Business:** E-Commerce – внешние транзакции с обменом ценностями; E-Business – более широкое понятие, включающее внутренние цифровые процессы.
*   **Влияние E-Commerce:** Позитивное (рост, инновации) и негативное (приватность, безопасность, доминирование гигантов).
*   **Восемь уникальных особенностей технологии E-Commerce:** Вездесущность, глобальный охват, универсальные стандарты, богатство, интерактивность, информационная плотность, персонализация/кастомизация, социальные технологии. (ОЧЕНЬ ВАЖНО!)
*   **Основные типы E-Commerce:** B2C, B2B, C2C, C2B, G2B, G2C – и их характеристики/примеры.
*   **Подтипы/Тренды:** M-commerce (мобильная), Social e-commerce (социальная), Local e-commerce (локальная) – их драйверы и особенности.
*   Понимание этих концепций важно для анализа современного цифрового бизнеса и его перспектив.

# Lesson 11: Audit and Controls of Information Systems

1.  **Зачем проводить аудит информационных систем (ИС)? (Слайд 2)**
    *   Чтобы убедиться, что **данные целостны и точны** (т.е. они правильные, полные и не испорчены).
    *   Чтобы проверить, насколько хорошо работают **меры безопасности**.
    *   Чтобы оценить, **соблюдаются ли правила и политики** организации и законодательства.

2.  **Что такое аудит ИС? (Слайд 3)**
    *   Это **независимая проверка (examination)** того, как работают **контроли** в информационных системах.
    *   **Цель:** Определить, являются ли эти контроли **достаточными (adequacy)** и **эффективными (effectiveness)**.
    *   Аудит охватывает все компоненты ИС: **аппаратное обеспечение (железо), программное обеспечение (софт), данные, людей и процедуры (правила)**.

3.  **Типы контролей в ИС (Слайд 4):**
    Контроли – это как меры предосторожности или "защитные механизмы".
    *   **Превентивные (Preventive) контроли:** Помогают **предотвратить** проблемы до того, как они возникнут.
        *   *Пример:* Пароли для входа в систему (контроль доступа).
    *   **Детективные (Detective) контроли:** Помогают **обнаружить** проблемы, если они уже произошли.
        *   *Пример:* Журналы событий (логи), которые записывают, кто и что делал в системе.
    *   **Корректирующие (Corrective) контроли:** Помогают **исправить** уже выявленные проблемы.
        *   *Пример:* Процедуры восстановления данных из резервной копии.

4.  **Общие контроли ИТ (IT General Controls - ITGCs) (Слайд 5):**
    Это "фундаментальные" контроли, которые влияют на всю ИТ-среду организации.
    *   **Операции дата-центра:** Как управляется физическое место, где стоят серверы.
    *   **Приобретение и обслуживание системного ПО:** Как покупают, устанавливают и обновляют операционные системы, СУБД и т.д.
    *   **Безопасность доступа:** Кто имеет доступ к системам и данным, и как этот доступ контролируется.
    *   **Разработка прикладных систем:** Как создаются и внедряются новые программы, чтобы они были безопасными и работали правильно.

5.  **Контроли приложений (Application Controls) (Слайд 6):**
    Это контроли, встроенные в конкретные программы (например, в бухгалтерскую систему), чтобы обеспечить правильность обработки данных.
    *   **Контроли ввода (Input Controls):** Проверяют данные, когда их вводят в систему.
        *   *Пример:* Проверка формата даты, проверка, что в поле "сумма" введены только цифры (validation checks).
    *   **Контроли обработки (Processing Controls):** Обеспечивают правильность обработки данных внутри системы.
        *   *Пример:* Сверка итоговых сумм между этапами обработки (run-to-run totals).
    *   **Контроли вывода (Output Controls):** Гарантируют, что результаты обработки верны и доставляются нужным людям.
        *   *Пример:* Журналы рассылки отчетов (distribution logs), проверка правильности отчетов.

6.  **Процесс аудита ИС (The IS Audit Process) (Слайд 7):**
    Аудит проходит в несколько этапов:
    1.  **Планирование (Planning):** Понимание системы, которую будут проверять, и определение целей аудита.
    2.  **Полевая работа (Fieldwork):** Сбор доказательств, тестирование контролей (проверка, как они работают на самом деле).
    3.  **Оценка (Evaluation):** Анализ полученных результатов, выявление недостатков и "пробелов" в контролях.
    4.  **Отчетность (Reporting):** Предоставление отчета с результатами аудита и рекомендациями по улучшению.
    5.  **Последующие действия (Follow-up):** Проверка, как выполняются рекомендации и устраняются недостатки.

7.  **Оценка рисков в аудите ИС (Risk Assessment in IS Audits) (Слайд 8):**
    Аудиторы не могут проверить абсолютно все, поэтому они фокусируются на самом важном.
    *   **Выявление (Identify) потенциальных рисков:** Что может пойти не так?
    *   **Оценка (Assess) влияния и вероятности:** Насколько серьезными будут последствия, если риск реализуется, и как часто это может произойти?
    *   **Приоритизация (Prioritize) фокуса аудита:** На какие области и контроли обратить внимание в первую очередь.

8.  **Инструменты и методы аудита (Tools & Techniques) (Слайд 9):**
    *   **Специальное аудиторское ПО (Audit software):** Программы типа ACL, IDEA, которые помогают анализировать большие объемы данных.
    *   **Интервью и наблюдения:** Беседы с сотрудниками и наблюдение за их работой.
    *   **Просмотр логов (Log reviews):** Анализ журналов событий системы.
    *   **Самооценка контролей (Control self-assessments):** Когда сотрудники сами оценивают эффективность контролей в своих подразделениях.

9.  **Примеры из реального мира (Real-World Examples) (Слайд 10):**
    Где применяется аудит ИС:
    *   **Банковские системы:** Проверка журналов транзакций.
    *   **Медицинские ИС:** Контроль доступа к медицинским картам пациентов.
    *   **Сайты электронной коммерции:** Аудит процесса обработки платежей.

10. **Аудит ИС в "полевых" проектах (IS Auditing in Field Projects) (Слайд 11):**
    Как это может касаться тебя (например, в учебных проектах):
    *   **Обеспечение** безопасности систем сбора данных.
    *   **Просмотр** системных логов во время анализа данных.
    *   **Оценка** рисков в рамках твоего проекта.

11. **Распространенные результаты аудита (Common Audit Findings) (Слайд 12):**
    Что аудиторы часто находят:
    *   **Слабый контроль доступа** (легко получить несанкционированный доступ).
    *   **Отсутствие процедур резервного копирования** (риск потери данных).
    *   **Неполная документация** (сложно понять, как система работает или должна работать).
    *   **Отсутствие процесса управления изменениями** (изменения в систему вносятся хаотично, без должного тестирования и одобрения).

12. **Резюме (Слайд 13):**
    *   Аудиты обеспечивают **надежность и безопасность** систем.
    *   Контроли – это ваш **набор инструментов** для достижения этого.
    *   Применяйте эти концепции в своих проектах.

---

**Ключевые моменты для запоминания из Лекции 11:**

*   **Аудит ИС** – это независимая проверка контролей для обеспечения целостности данных, безопасности и соответствия правилам.
*   **Три типа контролей:** превентивные (предотвратить), детективные (обнаружить), корректирующие (исправить).
*   **Два уровня контролей:** Общие ИТ-контроли (ITGC – фундамент) и контроли приложений (внутри конкретных программ).
*   **Процесс аудита:** Планирование -> Полевая работа -> Оценка -> Отчетность -> Последующие действия.
*   **Оценка рисков** помогает сфокусировать аудит на самом важном.
*   **Распространенные проблемы:** слабый доступ, нет бэкапов, плохая документация, нет управления изменениями.

# Lesson 12: Systems Development Life Cycle

1.  **Что такое Жизненный Цикл Разработки Систем (SDLC)? (Слайд 2)**
    *   Это **структурированный (упорядоченный) процесс** для разработки (создания) информационных систем. Представь, что это как подробный рецепт или план для постройки дома.
    *   Он помогает обеспечить **последовательность (consistency)** и **качество (quality)** создаваемой системы. То есть, все делается по определенным правилам, и результат получается хорошим.

2.  **Почему SDLC важен в "полевых" проектах (Field Projects)? (Слайд 3)**
    (Это, вероятно, относится к твоим учебным или практическим проектам)
    *   Помогает **управлять фазами (этапами)** проекта. Большой проект разбивается на более мелкие и понятные части.
    *   Способствует **документированию и планированию**. Заставляет все записывать и продумывать заранее, чтобы ничего не упустить.
    *   **Снижает риск провала** проекта. Если следовать плану, меньше шансов, что что-то пойдет не так.

3.  **Фазы (Этапы) SDLC (Слайд 4):**
    Это классические этапы, через которые проходит разработка системы. Они идут последовательно, но могут и повторяться в некоторых моделях.
    1.  **Планирование (Planning)**
    2.  **Анализ (Analysis)**
    3.  **Проектирование (Design)**
    4.  **Разработка (Development)**
    5.  **Тестирование (Testing)**
    6.  **Внедрение (Implementation / Deployment)**
    7.  **Сопровождение (Maintenance)**

4.  **Фаза Планирования (Planning Phase) (Слайд 5):**
    На этом этапе решают, *что* нужно сделать и *стоит ли* это делать.
    *   **Определение рамок (scope) и целей (objectives):** Что именно мы будем создавать? Каких результатов хотим достичь?
        *   *Пример:* Рамки – "создать сайт для заказа пиццы". Цель – "увеличить онлайн-заказы на 20%".
    *   **Проведение технико-экономического обоснования (feasibility study):** Оценивают, реально ли осуществить проект. Хватит ли денег (экономическая целесообразность)? Есть ли нужные технологии (техническая)? Сможем ли мы это сделать (операционная)?
    *   **Определение сроков (timeline) и ресурсов (resources):** Сколько времени это займет? Какие люди, деньги, инструменты нужны?

5.  **Фаза Анализа Систем (Systems Analysis) (Слайд 6):**
    На этом этапе подробно изучают, *что* должна делать система и *какие проблемы* она должна решать.
    *   **Сбор требований пользователей (user requirements):** Разговаривают с будущими пользователями системы, чтобы понять их нужды и пожелания. Что им нужно от системы?
    *   **Документирование текущих проблем:** Если есть старая система или процесс, описывают, что в нем не так.
    *   **Анализ потребностей и ограничений (constraints):** Какие функции абсолютно необходимы? Какие есть ограничения (бюджет, сроки, существующие технологии)?

6.  **Фаза Проектирования (Design Phase) (Слайд 7):**
    На этом этапе решают, *как* система будет выглядеть и работать.
    *   **Логическое проектирование (Logical design):** Описание системы "на бумаге" без привязки к конкретным технологиям. Что будет на входе (inputs), что на выходе (outputs), как будут двигаться данные (data flows).
    *   **Физическое проектирование (Physical design):** Конкретное описание того, как система будет реализована. Какая будет архитектура, какое оборудование (hardware), какой пользовательский интерфейс (interface).

7.  **Фаза Разработки (Development Phase) (Слайд 8):**
    На этом этапе происходит непосредственное создание системы.
    *   **Написание кода (Write code):** Программисты пишут программный код.
    *   **Создание базы данных (Build database):** Разрабатывается структура для хранения данных.
    *   **Интеграция компонентов (Integrate components):** Все части системы соединяются вместе, чтобы они работали как единое целое.

8.  **Фаза Тестирования (Testing Phase) (Слайд 9):**
    На этом этапе проверяют, правильно ли работает система и соответствует ли она требованиям.
    *   **Модульное (Unit) и системное (system) тестирование:** Проверяют отдельные маленькие части программы (модули), а затем всю систему целиком.
    *   **Исправление ошибок (Bug fixing):** Находят и исправляют ошибки ("баги") в программе.
    *   **Приемочное тестирование пользователем (User acceptance testing - UAT):** Будущие пользователи пробуют работать с системой и подтверждают, что она их устраивает и выполняет нужные функции.

9.  **Фаза Внедрения / Развертывания (Implementation / Deployment) (Слайд 10):**
    На этом этапе система запускается в работу.
    *   **Обучение пользователей (Train users):** Людей учат, как пользоваться новой системой.
    *   **Миграция данных (Migrate data):** Если были старые данные, их переносят в новую систему.
    *   **Запуск в эксплуатацию (Go live):** Система становится доступной для всех пользователей.

10. **Фаза Сопровождения и Поддержки (Maintenance & Support) (Слайд 11):**
    Этот этап начинается после запуска системы и может длиться годами.
    *   **Мониторинг производительности (Monitor performance):** Следят за тем, как система работает.
    *   **Применение обновлений (Apply updates):** Устанавливают новые версии, исправления безопасности.
    *   **Устранение проблем (Fix issues):** Исправляют ошибки, которые появляются уже во время работы.

11. **Модели SDLC (Слайд 12):**
    Существуют разные подходы (модели) к организации SDLC. Самые известные:
    *   **Водопадная (Waterfall):** Строго последовательное выполнение фаз. Закончил одну – переходишь к следующей. "Назад" вернуться сложно.
    *   **Гибкая (Agile):** Более гибкий подход. Работа ведется короткими циклами (спринтами), создаются небольшие работающие части системы, часто получается обратная связь от заказчика, и система адаптируется. Хорошо подходит, когда требования могут меняться.
    *   **Спиральная (Spiral):** Акцент на управлении рисками. Проект движется по спирали, на каждом витке повторяются этапы планирования, анализа рисков, разработки и оценки.
    *   **Итеративная (Iterative):** Сначала создается базовая версия системы, а затем она постепенно дорабатывается и расширяется новыми функциями в ходе нескольких итераций (повторений).

12. **Применение SDLC в "полевых" проектах (Слайды 13, 14):**
    Как использовать SDLC в твоих проектах:
    *   **Структурируй свой отчет** с использованием фаз SDLC (например, сделай разделы отчета по фазам).
    *   **Четко документируй** каждую фазу (описывай, что было сделано на каждом этапе).
    *   **Используй термины SDLC** для описания своего процесса.

---

**Ключевые моменты для запоминания из Лекции 12:**

*   **SDLC – это структурированный план** для создания информационных систем.
*   **Основные фазы SDLC:** Планирование -> Анализ -> Проектирование -> Разработка -> Тестирование -> Внедрение -> Сопровождение. Важно понимать, что происходит на каждой фазе.
*   SDLC помогает **управлять проектами, документировать и снижать риски**.
*   Существуют **разные модели SDLC** (Водопадная, Agile, Спиральная, Итеративная), каждая со своими особенностями.
*   Знание SDLC полезно для выполнения и описания **любых проектов по разработке систем**.
